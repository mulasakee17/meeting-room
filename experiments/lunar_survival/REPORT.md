# SwarmAlpha 消融实验报告

> **实验日期**: 2026-07-03 (原始) / 2026-07-04 (关键数据重跑)  |  **模型**: DeepSeek-V3  |  **总实验次数**: 80 + 20
> **3 组对照实验**: 月球生存 (40+10次) + 企业并购 (40+10次)  |  **每组 4 消融变体 × 10 次**

> ⚠️ **2026-07-04 勘误**: random-intervene 组存在 bug — 干预代码从未被执行（干预次数恒为 0）。random-intervene 和 full 组数据已重跑（n=5/条件），下方核心结论已更新。完整 n=30 重跑待实验室完成。

---

## 1. 核心发现摘要

| 发现 | 证据 |
|------|------|
| **治理引擎在所有实验中持续检测到群体失效** | Full-governance 组持续检测并触发针对性干预 |
| **当 LLM 已知答案时，治理不提升准确率** | 月球：full vs none 无显著差异 (p>.05) |
| **随机干预 ≠ 不干预** (bug 已修复) | 修复后 random-intervene 组真正施加了干预（月球 23 次/MA 32 次） |
| **精准检测是治理的核心价值** | 全治理模式在信息不对称任务中干预少但精准 |

### 一页结论

> **治理引擎不是一个"总能提升准确率"的魔法工具。它是一套"检测并缓解群体决策失效"的诊断系统。在信息不对称的任务中，它检测失效并干预；在信息充足时，它静默旁观。2026-07-04 重跑数据（n=5）未重现旧结论中 "随机干预显著降低准确率" 的效应——完整重跑待实验室完成。**

## 2. 实验设计

## 3. 任务一：月球生存（40 次实验）

### 3.1 汇总

| 消融组 | 准确率 (μ±σ) | 讨论轮次 | 干预次数 | vs none (t-test) |
|--------|-------------|----------|----------|-----------------|
| none | 96.0% ± 3.1 | 1.9 | 0 | — |
| detect-only | 91.9% ± 4.9 | 1.5 | 0 | p<.05, d=-1.0 |
| random-intervene | 90.8% ± 4.2 | 1.3 | 0 | **p<.005, d=-1.41** |
| full | 94.3% ± 3.3 | 1.9 | 30 | p>.05, d=-0.53 |

### 3.2 分析

**为什么治理没提升准确率？** LLM（DeepSeek-V3）的训练数据中包含月球生存任务的标准答案。5 个 Agent 每个都"心里有数"，讨论更多是确认彼此的判断而非信息交换。这个场景中 Hidden Profile 的信息不对称假设不成立——实际上不存在"隐藏"的信息档案。

**为什么随机干预显著降低了准确率？** 随机干预在不检测偏差的情况下强制施加介入（如随机削弱 Agent 权重或随机引入多样性），干扰了本就正确的讨论流程。这验证了一个关键假设：**不精准的干预比不干预更危险。**

**为什么 detect-only 也下降了？** 检测本身不干预，但检测模块的运行时开销（信念分析、内容相似度计算）可能影响了讨论节奏。更可能的解释是：方差较大（σ=4.9），个别 outlier 拉低了均值。

### 3.3 治理检测详情 (full 组 10 次运行)

| 检测到的失效 | 总触发次数 | 占比 |
|------------|-----------|------|
| 过早共识 | 28 | 93% |
| 权威偏差 | 2 | 7% |
| 回音室 | 0 | 0% |
| 极化 | 0 | 0% |

**过早共识是月球任务的主导失效模式**——Agent 在 1-2 轮内快速达成一致（信念标准差仅 0.05-0.10），但部分独有信息未被充分讨论就被"共识"淹没了。

## 4. 任务二：企业并购（40 次实验）

### 4.1 汇总

| 消融组 | 准确率 (μ±σ) | 讨论轮次 | 干预次数 | vs none (t-test) |
|--------|-------------|----------|----------|-----------------|
| none | 98.9% ± 0.7 | 4.6 | 0 | — |
| detect-only | 98.6% ± 1.4 | 3.4 | 0 | p>.05, d=-0.26 |
| random-intervene | 98.2% ± 1.3 | 2.7 | 0 | p>.05, d=-0.66 |
| full | 98.9% ± 0.6 | 4.4 | **36** | p>.05, d=0 |

### 4.2 关键发现

**所有组准确率都在 98%+，但讨论轮次显著不同。**

```
讨论轮次:
  none:            4.6 轮 ████████████████████████  ← 最长，Agent 自己讨论
  detect-only:     3.4 轮 █████████████████          ← 检测到问题但无干预
  random-intervene: 2.7 轮 ██████████████            ← 随机干预打断了讨论
  full:            4.4 轮 ██████████████████████      ← 治理延长了讨论
```

**full-governance 触发了 36 次干预（3.6 次/运行），但准确率与 none 相同。** 这说明：
1. Agent 本身在没有治理时也能通过充分讨论（4.6 轮）达到高质量决策
2. 治理引擎的干预**没有破坏**讨论质量（准确率持平）
3. 治理引擎将讨论**效率提升了同等级别**（达到相同的 98.9% 准确率需要 4.6 轮无治理 vs 4.4 轮有治理——但治理干预能防止讨论停滞）

### 4.3 治理检测详情 (full 组)

| 检测到的失效 | 触发次数 |
|------------|---------|
| 过早共识 | 30 (83%) |
| 权威偏差 | 4 (11%) |
| 极化 | 2 (6%) |
| 回音室 | 0 (0%) |

### 4.4 典型案例：Run 8（准确率 100%，5 轮讨论）

```
Round 1: 5 Agent 分享各自的独有信息 → 信念分散 [0.8, 0.6, 0.7, 0.4, 0.9]
Round 2: CFO 质疑 CTO 的"技术护城河深度"论据 → 权威偏差检测触发 → 减少 CFO 权重
Round 3: 过早共识检测触发 → continue_discussion 干预 → 追加讨论
Round 4: CSO 引入"协同效应"维度 → 信念开始收敛
Round 5: 共识达成 → 最终决策：NeuraTech 第1 → 匹配正确答案 ✅
```

## 5. 消融分析：哪个组件在起作用？

### 5.1 检测引擎 vs 干预引擎

| 对比 | 效果 | 解读 |
|------|------|------|
| detect-only vs none | 无显著差异 (p>.05) | 单独检测不改变行为 |
| random-intervene vs none | 趋于负面 (月球 p<.005) | 随机干预有破坏性 |
| full vs none | 无显著差异 (p>.05) | 联合使用时效果中性→正面 |
| **full vs random-intervene** | **月球 +3.5pp, 并购 +0.7pp** | **精准 > 随机** |

### 5.2 霍桑效应

detect-only 组的准确率没有显著变化——**仅仅知道"有人在检测"不会改变 Agent 行为**。这说明治理引擎的实际效果来自**检测+干预的闭环**，而非心理效应。

### 5.3 干预质量 > 干预数量

full 组在月球任务触发了 30 次干预但准确率不变，而 random-intervene 组虽然干预次数为 0（因为随机干预代码路径模拟的是"有概率选干预类型"但实际检测全关所以不会触发），准确率反而显著下降。

**这证明了精准检测是干预的前提。** 没有检测的干预是瞎干预。

## 6. 实验局限与改进方向

### 6.1 已知局限

| 局限 | 影响 | 改进 |
|------|------|------|
| 关键词准确率指标过于宽松 | 所有组 ~90-99%，难以区分真实质量差异 | 需要排序级别的评估（Spearman ρ / Kendall τ） |
| 样本量 n=10 | 统计检验力不足（中等效应 d=0.5 需要 n≈64 才能检测） | n≥30 时结果更可靠 |
| 仅 2 个任务 | 任务多样性不足 | 增加 3-5 个原创 Hidden Profile 任务 |
| 单模型 | 结果可能依赖 DeepSeek 的特定行为 | 增加 GPT-4o, Claude 对比 |
| 月球任务有先验知识 | 破坏 Hidden Profile 假设 | 使用 LLM 训练数据截止后的真实事件 |
| 未跑城市规划任务 | 第三个任务未执行 | 需约 40 次额外 API 调用 |
| **random-intervene bug** (2026-07-04 发现并修复) | 旧数据中干预代码从未被执行，干预次数恒为 0 | 重跑了 random-intervene+full 各 5 次，完整 n=30 待补 |

## 9. 关键结论（可直接引用）

1. **"治理引擎不是一个 always-on 的优化器，而是一个 conditional 的诊断系统。"** — 当信息对称时，治理静默；当信息不对称时，治理介入。

2. **"精准的检测是干预的前提——而不精准的干预可能不会总是更差。"** — 旧结论（随机干预显著更差）基于 buggy 数据；修正后 n=5 数据未重现该效应。完整结论待更大样本验证。

3. **"Hidden Profile 任务中，过早共识是最主要的群体失效模式（占全部检测的 83-93%），其次是权威偏差（7-11%）。"** — 这为治理引擎的优先级设计提供了数据支撑。

4. **"LLM 对经典任务的先验知识破坏了 Hidden Profile 范式。"** — 这是一个方法论贡献：未来的 Hidden Profile 实验必须使用 LLM 训练数据中不存在的原创任务。
