## 1. 逐例分类：三种失败模式

### 判定标准回顾

热力学引擎的结晶判定需同时满足三个条件：
- **R > 0.85**（序参量，信念方向一致性）
- **T < 0.20**（温度，信念方差归一化）
- **H < 0.35**（熵，5-bin Shannon 熵归一化）

连续 2 次结晶态 → 终止。强结晶（T < 0.10 **且** H < 0.10）→ 立即终止。

**诊断**：这是唯一一次系统**达到了结晶态但随后瓦解**的案例。Eval 6（28 次发言）：R=0.966, T=0.167, H=0.311 → 三条件全部满足 → crystallization count=1。只需再连续一次即可终止。但 eval 7 时，R 从 0.966 暴跌到 0.824（跌破 0.85 阈值），T 从 0.167 飙升到 0.396，H 从 0.311 反弹到 0.590。连续计数归零。

发生了什么？在 eval 3-5 期间（R>0.98, T<0.12, H=0.418），系统处于"准淬火态"——高同步但 H 不降。引擎可能在此期间向 agent 注入了淬火治理提示（"请从不同角度重新审视你的结论"）。这些提示的延迟效应在 eval 6-7 之间爆发，打散了原本脆弱的收敛。

**根因**：**淬火治理提示与收敛进程产生了对抗**。系统其实在缓慢而真实地收敛（H 从 0.418 → 0.311），但治理提示误判为"过早收敛"，注入的多样性提示反而破坏了进展。这暴露了一个根本性矛盾：**如何区分"真正在收敛但 H 下降慢"和"表面收敛（淬火态）"？**

⚠️ 另注：Run 8 的 `totalRounds=1451` 但 `totalUtterances=40`，这是数据异常（大约 36 轮空循环才产生 1 次有效发言），需要排查 `observeAgents` 是否有空返回路径。

## 3. 发言内容质量分析

> ⚠️ **数据限制**：实验 JSON 仅保存了 `thermoHistory` 和 `finalBeliefs`，未保存逐条发言内容。以下分析基于发言意愿公式的机制分析和热力学轨迹的间接推断。

### 3.1 发言意愿公式的质量盲区

当前 `content_driven` 的发言意愿公式：

```
raw = infoExposure × 0.6          ← 信息新颖性（权重最大）
    + beliefShift_bonus            ← 信念变动激励
    + consensusDeviation_bonus     ← 分歧激励
    + dependencyTriggered × 0.3    ← 依赖触发
    - recentlySpoke × 0.5          ← 抑制连续发言
```

**公式只回答"该不该发言"，不回答"发言有没有用"。** 存在 5 个质量问题：

#### 问题 1：infoExposure 奖励低价值信息

Agent 有 10 个独有关键词，其中 9 个无关、1 个关键。分享了关键的那 1 个后，infoExposure 仅下降 10%，agent 继续发言分享剩余 9 个无关关键词。**关键词的 relevance 未被建模。**

#### 问题 2：beliefShift 奖励不稳定（而非收敛）

讨论收敛时信念变动应减小——但公式中 beliefShift > 0.3 给 +0.4 的 bonus。这意味着信念还在剧烈振荡的 agent 更愿意发言，而已经收敛的 agent 被沉默。这恰好是 Run 2（τ=0.0）的情况：持续的信念振荡 → 持续的发言意愿 → 持续的噪音。

#### 问题 3：consensusDeviation 无法区分"有价值的异议"和"顽固的噪音"

持异议者有两种：(a) 有未被考虑的合理证据，(b) 纯粹的错误判断。公式给了两者相同的 bonus。在 Run 2 中，这可能导致一个认知偏差的 agent 持续获得发言权，阻止收敛。

#### 问题 4：权重静态，不适应讨论阶段

- **早期**（信息共享阶段）：infoExposure 应该权重最高 ✓
- **中期**（辩论阶段）：beliefShift 和 consensusDeviation 应该权重最高
- **晚期**（收敛阶段）：应该大幅降低所有发言意愿，只让有决定性证据的 agent 发言

当前公式在所有阶段使用相同权重，导致晚期仍在产生不必要的发言。

#### 问题 5：缺乏"发言质量"反馈回路

理想公式应包含：
```
utteranceQuality_bonus = α × wasReferencedByOthers   // 发言被他人引用
                       + β × changedOthersBeliefs     // 发言改变了他人信念
                       + γ × introducedVerifiedEvidence // 引入了可验证证据
```

这是一个 agent 声誉系统——产生有用信息的 agent 获得更多发言权，产生噪音的 agent 被降权。

### 3.2 从热力学轨迹推断的质量问题

#### 证据 A：Run 2（τ=0.0）— 纯粹的熵产生

41 次发言，H 在 0.31↔1.00 振荡，R 在 0.83↔0.96 振荡。系统像一个热机在做无用功——每次发言都产生熵，但从未降低自由能。这说明发言内容在引入新分歧而非解决旧分歧。

**如果发言质量高**：H 应呈下降趋势（信息消除不确定性），而非随机游走。

#### 证据 B：Run 8 的去结晶化 — 发言破坏收敛

Eval 6 系统已达结晶态，eval 7 瓦解。Eval 6→7 之间产生了约 3 次发言（28→31 utterances）。这 3 次发言：
- 要么引入了新的混淆信息
- 要么某个 agent 表达了强烈的反对意见（触发了共识偏差 bonus）
- 要么是淬火治理提示诱发了不必要的"重新审视"

**如果发言质量高**：已达结晶的系统不应被 3 次发言瓦解。

#### 证据 C：Run 1 的 H 平台 — 发言在做功但没改变 bin 分布

Run 1 在 eval 4-9（跨度 21 次发言，从第 19 到第 36 次）期间，R 从 0.821 升到 0.990，T 从 0.389 降到 0.089，但 H 始终 0.418。这意味着 agent 的信念数值在不断靠近（R↑, T↓），但保持在 2-3 个不同的 bin 中。

**这揭示了 H 指标的一个盲区**：bin 边界敏感。如果 5 个 agent 的信念分别是 [-0.1, 0.1, 0.3, 0.5, 0.7]，它们分布在 3 个 bin（间距 0.4），H≈0.45。但如果信念收敛到 [0.05, 0.08, 0.12, 0.15, 0.18]，它们全在同一个 bin，H=0。前者 τ 可能不低（排序方向正确）但 H 高。

### 3.3 发言质量维度的改进建议

```
发言意愿 = content_driven_score × quality_factor

quality_factor = 0.5 + 0.5 × agent_reputation
agent_reputation = f(
    past_utterances_referenced_count,   // 被引用次数
    past_belief_influence,              // 对他人的影响力
    past_evidence_verified,             // 证据被验证率
    consistency_score                   // 信念一致性（非随机游走）
)
```

首次发言的 agent quality_factor = 1.0（无罪推定），后续根据表现调整。连续产生噪音的 agent 的 quality_factor 衰减到 0.5，发言意愿减半。

### 深层问题（需重新思考架构）

热力学终止的核心矛盾：**R/T/H 测量的是信念分布的形状，而非信念的正确性。** Run 9 在 5 次发言后触发 strong_crystallized（H=0, T=0.07），但 τ 只有 0.4——系统快速收敛到了一个中等质量的答案。Run 1 的 τ=0.6（最佳）却被硬截断。

这表明信息聚合最有效的讨论（τ=0.6）不一定是最"热力学有序"的讨论。热力学隐喻（从混沌到结晶）假设秩序 = 质量，但 v2 实验显示这个假设不总成立。
