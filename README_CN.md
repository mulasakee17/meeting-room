## 核心发现

**治理 + 自适应是 LLM 多智能体系统的核心设计范式——但固定参数下无任何正面治理效果达到统计显著。2×2 因子设计（任务类型 × 讨论轮数）揭示轮次调节效应：3轮有限讨论中治理显示中等效应量（d=+0.65），5轮充分讨论后完全无效且反思干预显著有害（p=0.048）。洗牌对照是唯一显著正向发现。**

| | 投资决策（3轮·有限讨论, n=15） | 投资决策（5轮·充分讨论, n=15） | 企业并购（5轮·弱相互依赖） |
|---|---|---|---|
| **无治理 τ** | 0.422±0.344 | 0.778±0.325 | 0.533 |
| **完整治理 τ** | 0.644±0.344 | 0.778±0.325 | 0.613 |
| **净 Δτ** | **+0.133**（CI [−0.09, +0.35]） | **−0.089**（CI [−0.38, +0.21]） | **−0.123** |
| **d vs none** | +0.65 | +0.00 | +0.41 |
| **p 值** | 0.152（不显著） | 1.0（不显著） | 0.36（不显著） |
| **洗牌 τ** * | — | 1.000（p=0.264，不显著） | **0.900（p=0.0009，显著 ★）** |
| **显著有害干预** | — | **full_reflection p=0.048 ★** | — |
| **结论** | 中等效应量，方向为正 | 充分讨论后治理无效；反思干预显著有害 | 治理无效；洗牌是唯一显著正向发现 |

*\*洗牌对照（安慰剂检验 / 识别策略）：打乱智能体知识以破坏信息连贯性，同时保留讨论结构不变。在两个任务中，洗牌均未降低表现——反而提升。这表明信息整合的障碍不在于知识本身，而在于智能体对自有知识的过度自信。*

### 四条证据链支撑这一结论：

**1. 2×2 因子设计揭示轮次调节效应：治理在有限讨论中显示中等效应量。** 在投资任务的3轮有限讨论条件下（n=15），治理带来净 Δτ=+0.133（d=+0.65，p=0.152，CI [−0.09, +0.35]）的中等效应量改善——虽未达统计显著，但方向一致且效应量可观。然而，当讨论延长至5轮后（n=15），治理效果完全消失（d=+0.00，p=1.0），净 Δτ 甚至略负（−0.089，CI [−0.38, +0.21]）。这提示治理可能在信息整合不充分的边界条件下提供价值，但当智能体有足够时间自然整合信息后，治理的边际贡献趋零。2×2 因子设计（任务类型 × 讨论轮数）是本研究的关键方法学贡献——它系统性地揭示了治理效果的边界条件。

**2. 洗牌对照是唯一显著正向发现。** 在企业并购任务中，洗牌对照达到 τ=0.900（d=+1.80，p=0.0009）——全部 165 次实验中唯一达到统计显著的正向对比。打乱智能体知识破坏了专业过度自信，迫使智能体倾听他人，实现了更好的信息聚合。然而，完整治理本身并不显著（p=0.36），说明固定参数的定向干预尚不足以触发同样的信息整合效果。

**3. 首个统计显著的治理效果是有害的：full_reflection 在充分讨论中显著降低表现。** 在投资5轮任务中，`full_reflection`（τ=0.333，ΔQ=−22.2，p=0.048）达到统计显著——但方向为负。这是全部 165 次实验中首个统计显著的治理干预效果，却证明反思干预在相互依赖任务的充分讨论中是有害的。`full_weight`（τ=0.467，ΔQ=−15.6，p=0.173）也显示有害趋势。并购任务的单干预消融中，`full_diversity`（τ=0.660，p=0.174）、`full_weight`（τ=0.700，p=0.171）、`full_reflection`（τ=0.660，p=0.183）、`full_continue`（τ=0.620，p=0.267）均未达到统计显著。机制方向清晰（回声室检测 → 多样性注入），但固定剂量下效果不显著甚至有害。

**反直觉发现（并购洗牌）**：在弱相互依赖任务上，打乱智能体知识反而让表现超越了完整治理（τ=0.900 vs 0.613，p=0.0009）。为什么？并购任务中的智能体本身就掌握了全部 5 家公司的数据——他们不需要彼此也能做出合理判断。洗牌打破了他们的专业过度自信：CFO 手里拿的不再是财务数据而是陌生的技术数据，于是变得不那么确定，开始真正倾听他人的意见。结果是在没有任何治理干预的情况下实现了更好的信息聚合。这进一步强化了边界条件：在弱相互依赖任务上，治理不仅多余——打破过度自信（无论用什么手段，包括随机知识轮换）可以胜过定向干预。

> **核心洞察**：固定参数下，无任何正面治理效果达到统计显著——首个统计显著的治理效果反而是有害的（full_reflection, p=0.048）。治理的价值取决于任务结构和讨论充分度——2×2 因子设计系统揭示了这一边界条件。自适应阈值与剂量调整是将"方向性改善"推向"统计显著"、同时避免"显著有害"的关键下一步。

## SwarmAlpha 是什么？

SwarmAlpha 是用于生成上述证据的**治理运行时**——一个可嵌入的中间层，观察、检测并干预多智能体系统中的集体决策失效。

它**不创建**智能体，也**不管理**工作流。它通过插件形式接入现有框架，提供：

- 🔍 **观测**——从自然语言中提取智能体的信念和情绪
- 📊 **信念建模**——追踪信念演化与影响力传播
- 🚨 **偏差检测**——回声室、权威偏差、群体极化、过早共识
- 🛡️ **干预**——向智能体讨论中注入定向提示
- 📈 **评估**——五维评分，附 t 分布置信区间

**核心原则**：LLM 只负责感知（从语言中提取信念）。数学处理其余一切——共识计算、偏差检测、信念动力学。这意味着治理运行时可以作为一个**快速、廉价、可解释**的插件运行，零额外 LLM 调用。

## 为什么治理很重要？

当 5 个 AI 智能体讨论一个问题时，它们会陷入和人类群体相同的陷阱：

| 失效模式 | 表现 | 后果 |
|---------|------|------|
| **过早共识** | 第一轮就达成一致，关键信息从未被讨论 | 次优决策 |
| **权威偏差** | 一个过度自信的智能体主导全组 | 羊群效应 |
| **回声室** | 观点相似的智能体互相印证偏见 | 集体盲点 |
| **群体极化** | 分歧固化为僵局 | 决策瘫痪 |

**现有所有多智能体框架都不检测、也不干预这些失效。** SwarmAlpha 填补了这一空白——作为一个可插拔的治理层。

## 快速开始

### 1. 克隆并安装

```bash
git clone https://github.com/mulasakee17/meeting-room.git
cd meeting-room
npm install
```

### 2. 添加 API Key

```bash
cp .env.local.example .env.local
```

编辑 `.env.local`——至少添加一个 LLM API key：

```bash
# 必填：至少一个
DEEPSEEK_API_KEY=sk-your-key-here     # 从 https://platform.deepseek.com/ 获取
# OPENAI_API_KEY=sk-your-key-here      # 从 https://platform.openai.com/ 获取
# ANTHROPIC_API_KEY=sk-ant-your-key    # 从 https://console.anthropic.com/ 获取
```

**费用参考**：DeepSeek 约 0.07 元/次实验（5 智能体 × 5 轮）。OpenAI 约 0.7 元。Anthropic 约 1 元。

### 3. 运行

```bash
# Web 界面（演示模式无需 API key）
npm run dev                # → http://localhost:3000

# 运行实验（需要 API key）
npm run experiment          # 完整消融矩阵

# 分析结果（不需要 API key）
npm run analyze             # t 分布置信区间 + 置换检验 + 统计推断

# 参数敏感性分析（需要 API key）
npm run sensitivity         # 5 参数 × 5 值扫描

# 运行测试（不需要 API key）
npm test                    # 149 个测试
```

**演示模式**：打开 http://localhost:3000，点击"Run Comparison"——使用预计算场景，零 API 费用。"Live"模式发送真实 LLM 请求。

### 4. 作为 SDK 在你自己的项目中使用

```typescript
import { GovernanceRuntime, CustomAdapter } from "@/runtime";

const runtime = new GovernanceRuntime({
  maxRounds: 5,
  governanceMode: "full",           // "none" | "detect-only" | "full"
});

// 将你的智能体消息传入治理管线
const result = runtime.processRound(messages);

if (result.hasIntervention) {
  await applyInterventionToYourAgents(result.interventions[0]);
}

const evaluation = runtime.getSessionResult(finalDecision);
console.log(`决策质量：${evaluation.overallScore}/100`);
```

## 治理运行时

### 4 种治理模式 + 扩展消融

| 模式 | 检测 | 干预 | 用途 |
|------|------|------|------|
| `none` | ❌ | ❌ | 基线对照 |
| `detect-only` | ✅ | ❌ | 霍桑效应检验 |
| `random-intervene` | ❌ | ✅ 随机 | 消融实验："精准度是否必要？" |
| `full` | ✅ | ✅ 定向 | 生产使用 |
| **扩展模式** | | | |
| `shuffle` | ✅ | ✅ | 均值回归对照：打乱智能体知识 |
| `full_diversity` | 仅回声室 | 仅多样性 | 单干预消融 |
| `full_weight` | 仅权威偏差 | 仅削减权重 | 单干预消融 |
| `full_reflection` | 仅极化 | 仅反思 | 单干预消融 |
| `full_continue` | 仅过早共识 | 仅延长讨论 | 单干预消融 |

### 自适应治理

阈值和干预强度根据任务上下文自适应调整：

- **自适应阈值** 🔧 *已实现，尚未实验验证*：先跑一轮校准讨论 → 测量收敛速度、基础冗余度、影响力集中度 → 按任务自动缩放检测阈值
- **自适应剂量** 🔧 *已实现，尚未实验验证*：干预强度随偏差严重度、信息覆盖率和历史干预效果动态调整
- **交叉质证引擎** ✅ *已验证*：当智能体出现分歧，自动分为正反阵营，进行对抗性辩论，综合裁决并保留少数派报告

> **诚实声明**：本页所述 165 次实验均使用固定阈值和固定剂量。自适应阈值/剂量代码已实现但未与固定参数做实验对比。五维评估权重（0.20/0.25/0.20/0.17/0.18）为启发式设定，非经验校准——等权稳健性检查已规划。

### 五维决策评估

| 维度 | 测量的内容 | 权重 |
|------|-----------|------|
| **共识度** | Kuramoto 序参数 + 信念方差 + 收敛轨迹 | 20% |
| **可靠性** | 跨轮次 Cronbach's α + 交叉验证 + 可重复性 | 25% |
| **离散度** | 跨智能体信念/置信度方差 + 轮次波动 | 20% |
| **稳定性** | 轮次一致性 + 时序平滑度 | 17% |
| **影响力分析** | 基尼系数 + 网络中心性 + 影响路径 | 18% |

## 框架兼容性

SwarmAlpha **不绑定任何框架**。通过标准化适配器接口，它可以与任何多智能体系统协同工作：

| 框架 | 适配器 | 状态 |
|------|--------|------|
| **自建**（内置） | `CustomAdapter` | ✅ 完整集成 |
| **AutoGen**（微软） | `AutoGenAdapter` | 🔧 TypeScript 桥接（完整集成需 Python sidecar） |
| **CrewAI** | 计划中 | 🗓️ 路线图 |
| **LangGraph** | 计划中 | 🗓️ 路线图 |

每个适配器将框架原生消息翻译为标准的 `DiscussionMessage` 格式，并将治理干预回注到框架中。

### 可扩展检测与共享工具

治理引擎支持通过注册 API 添加**自定义偏差检测器**——无需修改核心引擎即可扩展检测能力：

```typescript
engine.registerDetector({
  type: "groupthink",
  detect(agentBeliefs, messages, config) {
    // 自定义检测逻辑
    return { detected: true, severity: "medium", description: "..." };
  },
});
```

共享工具模块（`src/lib/utils/`）消除了代码库中的重复代码：
- **`Registry<K,V>`** — 泛型注册表基类，供适配器/策略注册使用
- **`jsonUtils.ts`** — 统一 JSON 解析（stripCodeFences、safeJsonParse、extract 系列）
- **`statsUtils.ts`** — 统计工具（mean、std、variance、normalize）
- **`interventionPrompt.ts`** — 统一干预提示词格式化

## 实验证据

**165 次对照实验**（M&A 80 + 投资5轮 55 + 投资3轮 30）。主要指标：Kendall's τ + **组内 τ 轨迹（Δτ）**——追踪同一批智能体在多轮讨论中的变化。

### 为什么 Δτ + 洗牌对照很重要

| 方法 | 测量什么 | 陷阱 |
|------|---------|------|
| **Cohen's d**（组间） | 不同组之间的平均差异 | 不同智能体、不同初始条件 |
| **Δτ**（组内） | 同一批智能体跨轮次改进 | — |
| **洗牌对照** | 知识被打乱后的治理效果 | 检验均值回归 |

### 任务 1：相互依赖的投资决策（5轮，none/full n=15，其余 n=5）

没有单个智能体能独立得出正确答案。无治理 τ = 0.778。

| 消融模式 | τ（μ±σ） | Q（μ±σ） | d vs none | p 值 |
|----------|-----------|-----------|-----------|------|
| 无治理 | 0.778±0.325 | 89.0±16.1 | — | — |
| **完整治理** | **0.778±0.325** | **89.0±16.1** | +0.00 | 1.0（不显著） |
| 洗牌 | 1.000 | — | +0.77 | 0.264（不显著） |
| full_weight | 0.467 | ΔQ=−15.6 | — | 0.173（有害趋势） |
| full_reflection | 0.333 | ΔQ=−22.2 | — | **0.048（显著有害 ★）** |

- **完整治理 d=+0.00，p=1.0**——充分讨论后，治理无任何效果
- **洗牌 τ=1.000**（d=+0.77，p=0.264，n=5）——打乱知识反而达到满分，但未达显著（样本量不足）
- **full_reflection p=0.048（显著有害 ★）**——ΔQ=−22.2，全部 165 次实验中首个统计显著的治理干预效果，却证明反思干预在充分讨论中有害
- **full_weight 显示有害趋势**（p=0.173，ΔQ=−15.6）——在相互依赖任务上，削弱权威可能适得其反

### 任务 1b：投资决策边界条件（3轮，n=15，仅 none vs full）

| 消融模式 | τ（μ±σ） | Q（μ±σ） | d vs none | p 值 |
|----------|-----------|-----------|-----------|------|
| 无治理 | 0.422±0.344 | 71.3±17.2 | — | — |
| **完整治理** | **0.644±0.344** | **82.4±17.0** | +0.65 | 0.152（不显著，中等效应量） |

- **净 Δτ=+0.133，CI [−0.09, +0.35]**——方向性改善，CI 仍含 0 但更窄，未达显著（p=0.152）
- 3轮有限讨论中治理有中等效应量改善（d=+0.65，p=0.152），5轮充分讨论后效果消失（d=+0.00）且反思干预显著有害（p=0.048）——2×2 因子设计揭示轮次调节效应

### 任务 2：企业并购目标选择（5轮，n=15 for none/full，n=10 for others）

智能体可以独立推理。无治理 τ = 0.533。

| 消融模式 | τ（μ±σ） | Q（μ±σ） | d vs none | p 值 |
|----------|-----------|-----------|-----------|------|
| 无治理 | 0.533±0.209 | 76.7±10.5 | — | — |
| **完整治理** | **0.613±0.177** | **80.7±8.8** | +0.41 | 0.36（不显著） |
| **洗牌** | **0.900±0.194** | **95.0±9.7** | **+1.80** | **0.0009（显著 ★）** |
| full_diversity | 0.660±0.190 | — | — | 0.174（不显著） |
| full_weight | 0.700±0.316 | — | — | 0.171（不显著） |
| full_reflection | 0.660±0.190 | — | — | 0.183（不显著） |
| full_continue | 0.620±0.063 | — | — | 0.267（不显著） |

- **完整治理 d=+0.41，p=0.36**——统计上不显著
- **洗牌 d=+1.80，p=0.0009**——全部实验中唯一显著正向发现 ★。打乱知识打破专业过度自信，迫使智能体倾听
- **所有单干预消融均不显著**——固定参数下，单一机制不足以产生显著正向效果

### 边界条件（附证据）

| 论断 | 证据 |
|------|------|
| 2×2 因子设计揭示轮次调节效应 | 投资3轮 d=+0.65（p=0.152）→ 投资5轮 d=+0.00（p=1.0）|
| 治理在有限讨论中有中等效应量 | 投资3轮 净 Δτ=+0.133（d=+0.65），p=0.152，CI [−0.09, +0.35] 含 0 |
| 充分讨论后治理无效 | 投资5轮 d=+0.00，p=1.0，净 Δτ=−0.089 |
| 治理对弱相互依赖任务无效 | 并购 d=+0.41，p=0.36 |
| 首个显著治理效果是有害的 | 投资5轮 full_reflection p=0.048，ΔQ=−22.2 ★ |
| 洗牌对照是唯一显著正向发现 | 并购洗牌 d=+1.80，p=0.0009 |
| 单干预消融均未显著（并购） | 并购：full_diversity p=0.174，full_weight p=0.171，full_reflection p=0.183，full_continue p=0.267 |
| 部分干预在相互依赖任务上有害 | 投资5轮：full_reflection p=0.048（显著有害），full_weight p=0.173（有害趋势）|
| 在简单任务上打破过度自信胜过治理 | 并购洗牌 τ=0.900 > 完整治理 τ=0.613 |

**统计严格性**：t 分布 95% 置信区间 + 置换检验 p 值。2×2 因子设计（任务类型 × 讨论轮数）系统揭示治理效果的边界条件——这是本研究的关键方法学贡献。9 种消融模式。参数敏感性基础设施（5×5×5 扫描）。所有原始数据保存在 `experiments/v2/data*/`。

## 为什么这很重要

多智能体系统正被部署到高风险领域——金融、医疗、法律。当 5 个 AI 智能体讨论一个关键决策时，它们会犯和人类群体**相同的系统性错误**。现有框架（AutoGen、CrewAI、LangGraph）提供零治理能力。

SwarmAlpha 的 165 次对照实验揭示了：

1. **2×2 因子设计揭示轮次调节效应**——投资任务3轮条件下净 Δτ=+0.133（d=+0.65，p=0.152，中等效应量），但5轮充分讨论后效果消失（d=+0.00）
2. **治理有边界**——5轮充分讨论后效果消失；弱相互依赖任务上也不显著（p=0.36）
3. **首个显著治理效果是有害的**——投资5轮中 full_reflection p=0.048，显著降低表现（ΔQ=−22.2），证明固定参数干预可能适得其反
4. **洗牌对照是唯一显著正向发现**——并购任务洗牌对照 p=0.0009，提示信息整合机制有效，但固定参数的定向干预尚不足以触发同样效果
5. **自适应优化是下一步关键**——固定阈值/剂量下无任何正面治理对比显著，自适应阈值与剂量调整是将"方向性改善"推向"统计显著"的必经之路

**对 AI 部署的启示**：治理不是万能药——固定参数下甚至可能有害。先测量任务相互依赖性和讨论充分度。在信息整合不充分的边界条件下，治理可能提供方向性价值——但需配合自适应参数调优，以避免在充分讨论场景中触发"显著有害"的干预。在智能体已足够胜任或讨论已充分的场景中，治理的边际贡献趋零。

## 可扩展架构：从 5 个智能体到 500 个

SwarmAlpha 的讨论拓扑层让同一套治理引擎能在任意规模下运行：

| 规模 | 拓扑 | 行为 | 验证状态 |
|------|------|------|---------|
| **5 智能体** | `FlatTopology` | 圆桌讨论——所有智能体看到所有观点 | ✅ 165 次实验 |
| **40 智能体** | `GroupedTopology(8)` | 5 组 × 8 智能体，每轮重新洗牌——交叉授粉 | 🔧 已实现，未测试 |
| **500 智能体** | `CommitteeTopology` | 分组 → 代表 → 全体会议——联邦治理 | 🔧 已实现，未测试 |

治理引擎本身**在所有规模下保持不变**。只有讨论结构发生变化。偏差检测器和干预策略操作的是全局信念状态——它们不关心信念是在扁平讨论还是分组讨论中形成的。

> **诚实声明**：所有 165 次实验均使用 `FlatTopology`（5 智能体）。`GroupedTopology` 和 `CommitteeTopology` 已实现并通过单元测试，但尚未经过实验验证。

```typescript
// 一行配置扩展到 40 个智能体：
const engine = new DiscussionEngine({
  governanceMode: "full",
  topology: new GroupedTopology(8),  // ← 唯一需要改的地方
});
```

> *"不是构建智能体的框架，而是治理智能体的操作系统。"*
