## 核心发现

**治理在断裂环路修复后验证有效，扩样后统计确认。** Crisis 任务扩样至 n=24/cell 后，full vs none 大正效应（d=0.92，p=0.005，功效 88%），τ 从 0.408 提升到 0.617。洗牌对照仍最强（d=1.44，p<0.001）。跨任务验证（Supplier n=30）方向一致（d=0.47，p=0.089）。

### 2026-07-14 验证实验（治理环路修复后，扩样确认）

| | Crisis（n=24/cell） | Supplier（n=30/cell） |
|---|---|---|
| **无治理 τ** | 0.408 ± 0.182 | 0.680 ± 0.186 |
| **完整治理 τ** | 0.617 ± 0.263 | 0.767 ± 0.183 |
| **洗牌 τ** | **0.717 ± 0.243** | 0.697 ± 0.204 |
| **d vs none** | **+0.92**（p=0.005）/ **+1.44**（p<0.001） | **+0.47**（p=0.089）/ +0.09（ns） |
| **统计功效** | 88%（✅ 充分） | 43%（⚠️ 不足） |

**关键发现**：
1. **治理有效并统计确认**：Crisis d=0.92, p=0.005, 功效 88%——首次达到统计充分
2. **跨任务方向一致**：Supplier d=0.47, p=0.089（未达显著但方向一致，功效 43% 不足是主因）
3. **洗牌仍最强**（仅 Crisis）：d=1.44，打破专业知识垄断是最有效干预
4. **干预激活**（Crisis n=24）：89 次触发，47 次有效（52.8%）

> **关于洗牌对照**：shuffle 代表的是**信息交换的理论天花板**——所有 agent 获得全部专业知识、无垄断无信息壁垒时的决策质量上限。它不是 agent 协作的天花板，因为真实协作还涉及信任建立、观点博弈、共识协商等社交过程。shuffle 的作用是标定"如果信息完全流通能达到多好"，以此衡量治理在多大程度上逼近了信息整合的极限。

**设计改进**：
- 新任务消除信息泄露（agent 只掌握原始数据，正确答案需跨维度权衡）
- 修复 D1-D4 治理环路断裂
- 修复 API 失败假收敛
- 3 轮讨论留出治理空间

### 四条证据链

**1. 治理环路修复后验证有效（扩样确认）。** Crisis 任务 n=24/cell 显示 full vs none d=0.92（p=0.005，功效 88%），τ 提升 51%。

## 干预有效性与成本分析

基于 2026-07-14 Crisis 任务（full 模式 24 次实验）的 89 次干预，逐条追溯每个干预的目标 agent 信念变化与决策质量影响，并估算 token 成本。

### 有效率与决策质量

| 分类 | 次数 | 有效率 | 平均 τ 变化 | 说明 |
|------|------|--------|-------------|------|
| 有效干预 | 47 | 52.8% | **+0.265** | 信念变化 > 0.05 |
| 无效干预 | 42 | 47.2% | **+0.060** | 信念变化 ≤ 0.05，但不降低 τ |

- **有效干预确切提升决策质量**：τ 平均 +0.265
- **无效干预不降低决策质量**：τ 平均 +0.060（仍为正，不构成危害）

### 四类干预类型性价比

| 干预类型 | 次数 | 有效率 | 平均τ变化 | 平均token成本 | 结论 |
|----------|------|--------|-----------|--------------|------|
| force_reflection | 34 | **79.4%** | +0.222 | 1,295 | 最可靠 |
| reduce_weight | 31 | 61.3% | **+0.389** | 1,100 | 性价比最高 |
| introduce_diversity | 11 | 9.1% | +0.000 | 880 | 几乎无效 |
| continue_discussion | 13 | **0.0%** | **-0.400** | 1,025 | **有害** |

### 机制消融分析（新增）

按干预类型拆分的统计检验（vs 无干预基线 Δτ）：

| 干预类型 | Crisis d | Crisis p | Supplier d | Supplier p | 跨任务一致性 |
|----------|----------|----------|------------|------------|-------------|
| reduce_weight | **+1.51** | **0.0001** | +0.25 | 0.172 | ✅ 方向一致 |
| force_reflection | **+0.73** | **0.001** | +0.04 | 0.825 | ✅ 方向一致 |
| continue_discussion | -0.44 | 0.146 | — | — | 已禁用 |
| introduce_diversity | +0.20 | 0.527 | — | — | 已禁用 |

**关键发现**：reduce_weight 和 force_reflection 是驱动治理效果的核心机制。第3轮干预有效率 0%。

### 干预时机与有效性

| 轮次 | 干预次数 | 有效率 | 说明 |
|------|----------|--------|------|
| 第 1 轮 | 41 | 82.5% | 早期干预最有效 |
| 第 2 轮 | 18 | 62.2% | 仍然有效 |
| 第 3 轮 | 30 | 0.0% | 最后一轮无法改变结果 |

### Token 成本

- 总成本：75,680 tokens（约 5,045 tokens/实验）
- 有效成本：37,140 tokens（49.1%）
- 浪费成本：38,540 tokens（50.9%）
- 成本估算基于 [PromptInjector](./src/runtime/adapters/PromptInjector.ts) 中实际 prompt 文本长度 × 接收者数量

### 优化方案

停用 `continue_discussion` + `introduce_diversity` + 第 3 轮干预，可节省 **66.0%** 的治理 token 成本而不损失效果。

> **局限性**：每种条件 n=15，统计效力有限；token 成本为估算值非实际 API 计数；第 3 轮 0% 有效率部分受测量限制（无第 4 轮数据验证）；仅在单一任务类型上验证。

> **2026-07-14 实现状态更新**：上述"优化方案"已落地为代码默认行为——
> - `introduce_diversity` + `continue_discussion` 已默认禁用（[src/lib/governance/index.ts:188](./src/lib/governance/index.ts) `disabledInterventions`）
> - 最后一轮自动停止所有干预（[src/lib/governance/index.ts:688-692](./src/lib/governance/index.ts) `isLastRound` 拦截）
> - 真实 Token 追踪已替代估算（[experiments/v2/run.ts:109-113](./experiments/v2/run.ts) `tokenUsage` 字段记录 LLM API 实际计数）
> - 如需启用被禁用的干预，传 `disabledInterventions: []`

## 认知缺陷诊断与修复

> **2026-07-14 更新**：D1-D4 治理环路断裂已全部修复。新实验验证治理有效。

诊断识别出多 agent 讨论范式的 4 个根本性认知缺陷，并逐一修复：

| # | 认知缺陷 | 症状 | 修复 |
|---|---------|------|------|
| **1** | **状态感知缺失** | `buildPrompt` 未注入 agent 当前 `belief`/`confidence`，导致状态修改类干预（`reduce_weight`、`belief_perturbation`、`force_reflection`）对 LLM 不可见——模型"感受不到"干预。 | `buildPrompt` 现注入当前 belief/confidence 状态，使干预对 agent 可见。 |
| **2** | **无对话历史** | 所有 agent 看同一份全局摘要，没有任何 agent 知道"自己"上一轮说过什么。 | 个性化记忆：每个 agent 的 prompt 现包含（a）自己此前的发言 +（b）@提到自己的消息。 |
| **3** | **同步轮流念稿** | `Promise.all` 让一轮内所有 agent 并行发言，本轮互不可见。 | 顺序 `for` 循环：每个 agent 现能看到本轮已发言 agent 的累积上下文。 |
| **4** | **虚假影响力网络** | 用 belief 数值差"推断"agreement/disagreement/persuasion 边——产生的是幻影影响力图。 | 边**仅**由 agent 消息中显式的 `referencedAgents` 提及构建。 |

**含义**：修复后的环路闭合了此前打断干预反馈循环的 4 个缺口。在修复后环路下重跑时，边界条件结论可能发生变化——此前的"无效/有害"发现可能部分反映了干预从未真正到达 agent 感知这一事实。

## 硬伤修复（H 系列）

与认知缺陷诊断同步，识别并修复了一系列硬伤（H 系列）。此处记录以备溯源；本页 165 次实验的数字有部分是在这些修复落地之前生成的。

| ID | 缺陷 | 修复 |
|----|------|------|
| **H4** | Kuramoto 相位映射使用 `θ = π·b`，将极端极化（b=±0.99）映射到几乎相同的相位（R≈1）——误判为共识。 | 修正为 `θ = (π/2)·b`。现在 b=±0.99 → R≈0（真正极化），b=0 → R=1（真正共识）。 |
| **H6** | `convergenceSpeed` 代码注释错误（公式方向正确）。 | 注释纠正；公式不变。 |
| **H2** | `ablationModes` 仅有 `["none","full"]`（2 模式 × 15 = 30 次）。 | 扩展为 7 种完整模式：`none / full / shuffle / full_diversity / full_weight / full_reflection / full_continue`。完整设计现 7 × 15 = 105 次（待实验室执行）。 |
| **H19** | `introduceDiversity` 使用 `Math.random()`，导致干预跨运行不可复现。 | 替换为 `mulberry32` 种子化 PRNG——给定种子下干预现可确定复现。 |
| **H17** | 缓存污染：失败运行留下的错误占位文件被后续运行拾取。 | 删除污染占位文件；受影响实验从干净状态重跑。 |
| **H18** | `interventionPrompt` 在策略文件和 `PromptInjector` 中内联不一致。 | 统一 `interventionPrompt` 助手接入全部 8 处调用点（4 策略文件 + `PromptInjector` 4 处）。 |

> **Kuramoto 公式更新（H4）**：文档/代码中凡出现 Kuramoto 相位映射处，公式现为 `θ = (π/2) · b`（原 `θ = π · b`）。这是实质性修复，而非表面调整——它改变了对极化状态的共识检测。

## SwarmAlpha 是什么？

SwarmAlpha 是用于生成上述证据的**治理运行时**——一个可嵌入的中间层，观察、检测并干预多智能体系统中的集体决策失效。

**应用场景**：为 LLM 多 agent 协作决策提供实时过程治理——在共识形成过程中检测极化、权威偏置、回声室、过早共识四类偏差并施加干预，确保少轮次讨论的决策质量。

它**不创建**智能体，也**不管理**工作流。它通过插件形式接入现有框架，提供：

- 🔍 **观测**——从自然语言中提取智能体的信念和情绪
- 📊 **信念建模**——追踪信念演化与影响力传播
- 🚨 **偏差检测**——回声室、权威偏差、群体极化、过早共识
- 🛡️ **干预**——向智能体讨论中注入定向提示
- 📈 **评估**——五维评分，附 t 分布置信区间
- 🧪 **因果效应估计**——最近邻轨迹匹配 + 置换检验，估计干预的反事实效应

**核心原则**：LLM 只负责感知（从语言中提取信念）。数学处理其余一切——共识计算、偏差检测、信念动力学。这意味着治理运行时可以作为一个**快速、廉价、可解释**的插件运行，近乎零额外 LLM 调用（仅当 agent 未输出 `[GOV]` 结构化标签时，`StateInferenceBridge` 回退到 LLM 推断）。

## 为什么治理很重要？

当 5 个 AI 智能体讨论一个问题时，它们会陷入和人类群体相同的陷阱：

| 失效模式 | 表现 | 后果 |
|---------|------|------|
| **过早共识** | 第一轮就达成一致，关键信息从未被讨论 | 次优决策 |
| **权威偏差** | 一个过度自信的智能体主导全组 | 羊群效应 |
| **回声室** | 观点相似的智能体互相印证偏见 | 集体盲点 |
| **群体极化** | 分歧固化为僵局 | 决策瘫痪 |

**现有所有多智能体框架都不检测、也不干预这些失效。** SwarmAlpha 填补了这一空白——作为一个可插拔的治理层。

## 跨任务验证（2026-07-14，扩样确认）

为验证核心发现的**跨任务普适性**（而非 Crisis 任务特异），新增第 2 个任务：**核心零部件供应商选择**（5 供应商 × 5 维度的隐藏档案）。该任务与 Crisis 任务结构相同（5 方案 × 5 维度隐藏档案）但场景完全不同，完美对照。两任务均已完成扩样（Crisis n=24，Supplier n=30）。

### 双任务对比

| 指标 | Crisis（n=24/cell） | Supplier（n=30/cell） | 跨任务一致性 |
|------|------------------|--------------------|-------------|
| **none τ** | 0.408 ± 0.182 | 0.680 ± 0.186 | — |
| **full τ** | 0.617 ± 0.263 | 0.767 ± 0.183 | — |
| **shuffle τ** | 0.717 ± 0.243 | 0.697 ± 0.204 | ⚠️ 任务依赖 |
| **治理 Δτ** | **+0.209** | **+0.087** | ✅ 方向一致 |
| **治理 d** | 0.92 (p=0.005) | 0.47 (p=0.089) | ✅ 方向一致 |
| **统计功效** | 88% ✅ | 43% ⚠️ | — |
| **共识-质量 r** | -0.137 | -0.107 | ✅ 都≈0 |

### 跨任务验证的发现

**1. 治理效果方向跨任务一致**：两个任务都是 full > none，说明治理不是 Crisis 任务特异的现象。Crisis 已达统计确认（p=0.005），Supplier 方向一致但功效不足（需 n=72 达 80% 功效）。

**2. "虚假共识"现象跨任务普适**：两个任务的共识度与质量相关都接近零（r=-0.14 vs r=-0.11），证明"高共识≠高质量"是 LLM multi-agent 的普遍特征。

**3. Shuffle 对照的边界条件**（意外发现）：Crisis 中 shuffle 显著优于 none（d=1.44, p<0.001），但 Supplier 中 shuffle 无效应（d=0.09, p=0.78）。原因分析：
   - **基线难度差异**：Crisis none τ=0.41（低基线，任务困难），shuffle 通过信息整合大幅提升；Supplier none τ=0.68（高基线，任务较易），shuffle 无提升空间
   - **天花板效应**：Supplier 基线已接近 full 水平（0.68 vs 0.77），信息整合的边际收益趋零
   - **结论**：shuffle 对照的有效性受任务难度调节——在困难任务中显著，在容易任务中因天花板效应而不适用

**4. 机制消融跨任务一致**：reduce_weight 和 force_reflection 在两任务中均 d>0（方向一致）。Crisis 中两者均显著（reduce_weight d=1.51 p=0.0001, force_reflection d=0.73 p=0.001），Supplier 中方向一致但不显著。

### 学术意义

跨 2 个独立任务、**161 次实验**、3 种条件，核心发现**方向一致**：
- ✅ 治理有效（Crisis 统计确认 d=0.92 p=0.005，Supplier 方向一致 d=0.47）
- ✅ 虚假共识存在（r≈0 跨任务复现）
- ✅ 反思+减权 > 多样性（干预类型差异跨任务一致）
- ⚠️ Shuffle 对照有边界条件（任务难度依赖）

### 自适应治理

阈值和干预强度根据任务上下文自适应调整：

- **自适应阈值** 🔧 *已实现，尚未实验验证*：先跑一轮校准讨论 → 测量收敛速度、基础冗余度、影响力集中度 → 按任务自动缩放检测阈值
- **自适应剂量** 🔧 *已实现，尚未实验验证*：干预强度随偏差严重度、信息覆盖率和历史干预效果动态调整
- **F 分解驱动的干预优先级排序** ✅ *已实现 + 回测验证 + 假设证伪修正*：当多个检测器同时触发时（Crisis 实验中 91.7% 的场景），按社会自由能 F = (1-R) + T·H 的分解项对干预排序。回测（97 次 force_reflection 事件，p=0.041）**证伪**了原假设 `force_reflection↔结构性无序`——force_reflection 实为*降噪*干预（热性主导时有效，极化时有害），已修正映射为 `thermal·(1-structural)`。`reduce_weight↔热性无序` 方向支持但不显著（p=0.100）。详见 [THERMODYNAMICS_INTEGRATION.md §5.4](./THERMODYNAMICS_INTEGRATION.md)
- **交叉质证引擎** ✅ *已实现 + 单元测试*：当智能体出现分歧，自动分为正反阵营，进行对抗性辩论，综合裁决并保留少数派报告

> **诚实声明**：本页所述 165 次实验均使用固定阈值和固定剂量。自适应阈值/剂量代码已实现但未与固定参数做实验对比。五维评估权重（0.20/0.25/0.20/0.17/0.18）为启发式设定，非经验校准——等权稳健性检查已规划。

## 实验证据

**165 次历史对照实验**（M&A 80 + 投资5轮 55 + 投资3轮 30）+ **161 次扩样实验**（Crisis 72 + Supplier 89）。主要指标：Kendall's τ + **组内 τ 轨迹（Δτ）**——追踪同一批智能体在多轮讨论中的变化。

> **消融设计更新（H2）**：`ablationModes` 已从 `["none","full"]` 扩展为 7 种完整模式（`none / full / shuffle / full_diversity / full_weight / full_reflection / full_continue`）。完整 7 模式实验矩阵（105 次，7 × 15）待实验室执行；下方 165 次实验数字在此扩展前生成，按原样保留以备溯源。

### 为什么 Δτ + 洗牌对照很重要

| 方法 | 测量什么 | 陷阱 |
|------|---------|------|
| **Cohen's d**（组间） | 不同组之间的平均差异 | 不同智能体、不同初始条件 |
| **Δτ**（组内） | 同一批智能体跨轮次改进 | — |
| **洗牌对照** | 知识被打乱后的治理效果 | 检验均值回归 |

### 任务 1：相互依赖的投资决策（5轮，none/full n=15，其余 n=5）

没有单个智能体能独立得出正确答案。无治理 τ = 0.778。

| 消融模式 | τ（μ±σ） | Q（μ±σ） | d vs none | p 值 |
|----------|-----------|-----------|-----------|------|
| 无治理 | 0.778±0.325 | 89.0±16.1 | — | — |
| **完整治理** | **0.778±0.325** | **89.0±16.1** | +0.00 | 1.0（不显著） |
| 洗牌 | 1.000 | — | +0.77 | 0.264（不显著） |
| full_weight | 0.467 | ΔQ=−15.6 | — | 0.173（有害趋势） |
| full_reflection | 0.333 | ΔQ=−22.2 | — | **0.048（显著有害 ★）** |

- **完整治理 d=+0.00，p=1.0**——充分讨论后，治理无任何效果
- **洗牌 τ=1.000**（d=+0.77，p=0.264，n=5）——打乱知识反而达到满分，但未达显著（样本量不足）
- **full_reflection p=0.048（显著有害 ★）**——ΔQ=−22.2，全部 165 次实验中首个统计显著的治理干预效果，却证明反思干预在充分讨论中有害
- **full_weight 显示有害趋势**（p=0.173，ΔQ=−15.6）——在相互依赖任务上，削弱权威可能适得其反

### 任务 1b：投资决策边界条件（3轮，n=15，仅 none vs full）

| 消融模式 | τ（μ±σ） | Q（μ±σ） | d vs none | p 值 |
|----------|-----------|-----------|-----------|------|
| 无治理 | 0.422±0.344 | 71.3±17.2 | — | — |
| **完整治理** | **0.644±0.344** | **82.4±17.0** | +0.65 | 0.152（不显著，中等效应量） |

- **净 Δτ=+0.133，CI [−0.09, +0.35]**——方向性改善，CI 仍含 0 但更窄，未达显著（p=0.152）
- 3轮有限讨论中治理有中等效应量改善（d=+0.65，p=0.152），5轮充分讨论后效果消失（d=+0.00）且反思干预显著有害（p=0.048）——2×2 因子设计揭示轮次调节效应

### 任务 2：企业并购目标选择（5轮，n=15 for none/full，n=10 for others）

智能体可以独立推理。无治理 τ = 0.533。

| 消融模式 | τ（μ±σ） | Q（μ±σ） | d vs none | p 值 |
|----------|-----------|-----------|-----------|------|
| 无治理 | 0.533±0.209 | 76.7±10.5 | — | — |
| **完整治理** | **0.613±0.177** | **80.7±8.8** | +0.41 | 0.36（不显著） |
| **洗牌** | **0.900±0.194** | **95.0±9.7** | **+1.80** | **0.0009（显著 ★）** |
| full_diversity | 0.660±0.190 | — | — | 0.174（不显著） |
| full_weight | 0.700±0.316 | — | — | 0.171（不显著） |
| full_reflection | 0.660±0.190 | — | — | 0.183（不显著） |
| full_continue | 0.620±0.063 | — | — | 0.267（不显著） |

- **完整治理 d=+0.41，p=0.36**——统计上不显著
- **洗牌 d=+1.80，p=0.0009**——全部实验中唯一显著正向发现 ★。打乱知识打破专业过度自信，迫使智能体倾听
- **所有单干预消融均不显著**——固定参数下，单一机制不足以产生显著正向效果

### 边界条件（附证据）

| 论断 | 证据 |
|------|------|
| 2×2 因子设计揭示轮次调节效应 | 投资3轮 d=+0.65（p=0.152）→ 投资5轮 d=+0.00（p=1.0）|
| 治理在有限讨论中有中等效应量 | 投资3轮 净 Δτ=+0.133（d=+0.65），p=0.152，CI [−0.09, +0.35] 含 0 |
| 充分讨论后治理无效 | 投资5轮 d=+0.00，p=1.0，净 Δτ=−0.089 |
| 治理对弱相互依赖任务无效 | 并购 d=+0.41，p=0.36 |
| 首个显著治理效果是有害的（未校正） | 投资5轮 full_reflection p=0.048（未校正；Bonferroni 校正后不显著），ΔQ=−22.2 ★ |
| 洗牌对照是唯一显著正向发现 | 并购洗牌 d=+1.80，p=0.0009 |
| 单干预消融均未显著（并购） | 并购：full_diversity p=0.174，full_weight p=0.171，full_reflection p=0.183，full_continue p=0.267 |
| 部分干预在相互依赖任务上有害 | 投资5轮：full_reflection p=0.048（未校正显著有害；Bonferroni 校正后不显著），full_weight p=0.173（有害趋势）|
| 在简单任务上打破过度自信胜过治理 | 并购洗牌 τ=0.900 > 完整治理 τ=0.613 |

**统计严格性**：t 分布 95% 置信区间 + 置换检验 p 值（含 Bonferroni/BH FDR 多重比较校正）。部分因子设计（任务类型 × 讨论轮数 × 治理模式）系统揭示治理效果的边界条件——这是本研究的关键方法学贡献。9 种总配置（4 种治理模式 + 5 种扩展消融）。参数敏感性基础设施（5×5×5 扫描）。所有原始数据保存在 `experiments/v2/data*/`。

## 因果效应估计（轨迹匹配）

超越相关性分析（t 检验、置换检验），SwarmAlpha 引入**因果效应估计模块**——回答"干预导致最终 τ 变化了多少，相比无干预的反事实？"

**方法**：最近邻轨迹匹配（k=5）+ 逆距离加权反事实估计 + 10000 次置换检验 + 10000 次 Bootstrap CI。对每个处理实验，从 None 基线池中按第 1 轮轨迹距离选 5 个最近邻 donor，反事实 τ = 匹配 donor 最终 τ 的加权平均。距离公式：加权欧氏（τ 0.5 + 信念多样性 0.3 + 信念均值 0.2）。Cohen's d 钳制到 [−10, 10] 以处理小样本不稳定。

在 165 次实验数据上的关键发现（注意：数据早于 2026-07-12 治理环路修复，state-modification 类干预效应可能被低估）：

| 分组 | n_trt | 观测 τ | 反事实 τ | 效应 | 95% CI | d | p |
|------|-------|--------|----------|------|--------|---|---|
| 投资 3 轮 | 15 | 0.600 | 0.407 | +0.193 | [+0.01, +0.37] | 0.69 | 0.199 |
| 投资 5 轮 | 15 | 0.565 | 0.677 | −0.111 | [−0.27, +0.04] | −0.49 | 0.414 |
| 并购 5 轮 | 15 | 0.613 | 0.478 | +0.135 | [+0.07, +0.20] | 0.96 | 0.067 |

- **并购 5 轮**：因果效应 +0.135，95% CI [+0.07, +0.20] 不含 0，d=0.96（大效应），p=0.067——所有分析中最接近显著的结果。**注：CI（bootstrap 百分位）与 p（置换检验）来自不同方法；CI 排除 0 而 p>0.05 的差异反映方法学差异，非矛盾。** 仅 1 个预干预期，此为提示性而非结论性因果证据。
- **投资 3 轮**：效应 +0.193，CI 下界刚好 >0，d=0.69（中等）——与边界条件假设一致
- **投资 5 轮**：效应 −0.111，方向为负——与"多轮治理有害"发现一致

**按干预类型分解**：`continue_discussion` 与 `introduce_diversity` 显示正向效应；`force_reflection` 在投资 5 轮任务上有害。

**方法学局限**：预干预期短（仅 1 轮轨迹）、小样本（n=15/cell）、SUTVA 假设、历史数据存在治理环路断裂。修复后重跑实验以寻求 p<0.05 的显著因果证据为下一步。

> 运行：`npx tsx experiments/v2/causalAnalysis.ts`（不需要 API key，使用已有实验数据）。

## 为什么这很重要

多智能体系统正被部署到高风险领域——金融、医疗、法律。当 5 个 AI 智能体讨论一个关键决策时，它们会犯和人类群体**相同的系统性错误**。现有框架（AutoGen、CrewAI、LangGraph）提供零治理能力。

SwarmAlpha 的 326 次对照实验（历史 165 + 扩样 161）揭示了：

1. **治理环路修复后统计确认有效**——Crisis 任务 n=24/cell 显示 d=0.92, p=0.005, 功效 88%，治理从"方向性"升级为"统计确认"
2. **跨任务方向一致**——Supplier（n=30）d=0.47, p=0.089，方向一致但功效不足（需 n=72 达 80%）
3. **机制消融揭示驱动因素**——reduce_weight（d=1.51, p=0.0001）和 force_reflection（d=0.73, p=0.001）是核心驱动
4. **洗牌对照有边界条件**——Crisis 中 d=1.44（p<0.001），Supplier 中无效应（d=0.09）——受任务难度调节
5. **干预时机关键**——第1轮有效率 82.5%，第3轮 0%；已禁用有害干预，节省 66% 成本

**对 AI 部署的启示**：治理不是万能药——固定参数下甚至可能有害。先测量任务相互依赖性和讨论充分度。在信息整合不充分的边界条件下，治理可能提供方向性价值——但需配合自适应参数调优，以避免在充分讨论场景中触发"显著有害"的干预。在智能体已足够胜任或讨论已充分的场景中，治理的边际贡献趋零。

## 可扩展架构：从 5 个智能体到 500 个

SwarmAlpha 的讨论拓扑层让同一套治理引擎能在任意规模下运行：

| 规模 | 拓扑 | 行为 | 验证状态 |
|------|------|------|---------|
| **5 智能体** | `FlatTopology` | 圆桌讨论——所有智能体看到所有观点 | ✅ 165 次实验 |
| **40 智能体** | `GroupedTopology(8)` | 5 组 × 8 智能体，每轮重新洗牌——交叉授粉 | 🔧 已实现，未测试 |
| **500 智能体** | `CommitteeTopology` | 分组 → 代表 → 全体会议——联邦治理 | 🔧 已实现，未测试 |

治理引擎本身**在所有规模下保持不变**。只有讨论结构发生变化。偏差检测器和干预策略操作的是全局信念状态——它们不关心信念是在扁平讨论还是分组讨论中形成的。

> **诚实声明**：所有 165 次实验均使用 `FlatTopology`（5 智能体）。`GroupedTopology` 和 `CommitteeTopology` 已实现并通过单元测试，但尚未经过实验验证。

```typescript
// 一行配置扩展到 40 个智能体：
const engine = new DiscussionEngine({
  governanceMode: "full",
  topology: new GroupedTopology(8),  // ← 唯一需要改的地方
});
```

> *"不是构建智能体的框架，而是治理智能体的操作系统。"*
