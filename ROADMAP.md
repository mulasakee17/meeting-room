# SwarmAlpha 发展路线图

## 战略定位

> **不做多智能体框架，做多智能体社会的治理操作系统。**
>
> 不做"决策优化工具"，做"群体过程的可观测性基础设施"。

竞争优势：
- 所有人的注意力在"如何构建 agent"和"如何让 agent 协作"——AutoGen、CrewAI、LangGraph
- 没人在做"agent 协作过程中出了什么问题"和"如何治理"
- SwarmAlpha 的核心引擎已实现、已审计、零耦合——先发优势

## Phase 2：过程监控演示（1-2 周）

**目标：不依赖 τ，展示治理引擎的独立价值**

### 2.1 静态演示：病例分析
取一次真实实验的完整对话记录，做"过程审计报告"：
- 第 1 轮：回声室检测触发（3/5 agent 已有相同观点）→ 即使最终 τ=1.0，过程是有缺陷的
- 第 3 轮：CEO 的发言被引用 7 次，其他 agent 被引用 0-2 次 → 权威偏差存在
- 展示：如果当时有 diversity 干预，讨论会更快覆盖缺失信息

这份报告的价值：**证明引擎能看见人类看不见的东西**。τ 是滞后指标——决策做完才知道对不对。检测是实时指标——决策过程中就能发现问题。

### 2.2 Web UI 决策审计视图
在现有的 Demo 页面上加一个 tab："Decision Audit"。
- 左栏：讨论时间线（每轮每人的发言）
- 中栏：信念演化图（每人 belief 随时间变化）
- 右栏：检测事件线（何时触发了什么偏差检测）
- 不要求实时——静态回放即可

## Phase 3：智能体社会模拟（2-4 周）

**目标：扩展实验范式的边界——从 5 agent 决策到 50+ agent 社会动态**

这阶段是**与教授合作的最佳敲门砖**——"智能体社会"比"决策优化"更前沿，且完全避开 τ 天花板。

### 3.1 信息传播与回声室实验
- **场景**：50 agent，社交媒体拓扑（关注/粉关系），一个"新闻事件"在不同子群中传播
- **变量**：有无治理干预（diversity 注入打破信息茧房）
- **度量**：不是 τ——是信息覆盖率、信念极化度、回声室持续时间
- **成本**：50 agent × 1 轮（单向转发）≈ 无 LLM 调用——可以纯模拟

### 3.2 组织结构对比实验
- **场景**：100 agent 完成同一个任务，对比三种拓扑（Flat / Grouped / Committee）
- **度量**：决策速度、信息损失、影响力集中度
- **关键**：Committee 拓扑有信息瓶颈但可扩展——探索"效率 vs 质量"的权衡
- **成本**：可以用规则 agent（非 LLM）做大规模，LLM agent 做小规模验证

### 3.3 治理结构的比较制度分析
- **场景**：同一个 20-agent 社会，测试不同治理制度（无治理 / 多数投票 / 专家委员会 / SwarmAlpha 自适应）
- **度量**：长期满意度、少数意见存活率、决策切换频率
- **学术价值**：这是政治科学+AI 的交叉——比较制度分析在 agent 社会中的应用

原结论 C1（force_reflection "反向强化 +0.68"）因**归因完全混淆**（100% 样本同轮有 reduce_weight）而撤回。这是项目自我纠错机制的体现，但也暴露了早期分析中归因不严谨的问题。修正后的部分隔离分析（F9：n=5，5/5 上升 +0.94）方向与原结论一致，但样本量更小、仍非严格因果隔离。

### 新发现 F8-F10 的诚实定性

- **F8（失败成本 2.9x）**：n=6（成功 4 + 失败 2），样本量小，但方向清晰
- **F9（force_reflection 反向强化）**：n=5，极小样本，"部分隔离"仍非严格因果隔离
- **F10（reduce_weight 72% 压制）**：n=18，相对可靠，但仍是观察性数据

### 项目水平从 5.42 → 6.90 的诚实性

提升是真实的，但路径不是"写代码→变强"而是"搜索外部世界→发现真空间隙"。6.20 之前的提升（B1-B8 + A3）来自内部补课；6.20 → 6.90 的提升来自**外部定位**——A2A 协议标准化恰好为发言治理层留出了 1-2 年的窗口期。这不是 SwarmAlpha 变强了，而是外部世界的发展验证了它的方向。

但 6.90 仍是"有待验证的学术工作"。发言意愿公式的权重未经校准、因子集可能不完整、仅在 DeepSeek-V3 + FlatTopology 上验证过。要突破 7.0（一流本科生科研的门槛），文献对齐和形式化分析是不够的——需要**跨模型验证**来证明这些发现不是模型特异，或需要**发言策略消融实验**来证明 content_driven 的 5 因子优于替代方案。

# 附录 B：项目整体评价

> 本节原为独立文档 PROJECT_EVALUATION.md，现已合并入 ROADMAP.md 以集中规划信息。

## 一句话总评

**一个统计严谨、方法学有原创性、但尚未完成跨模型/跨拓扑验证的 LLM 多智能体治理研究项目。** 其最强的三个发现（虚假共识 r≈0、治理药物动力学、shuffle 信息上限方法学）在当前实验条件下统计确定。最突出的问题是单模型（仅 DeepSeek-V3）和单拓扑（仅 FlatTopology）限制了发现的普适性。

---

## 一、分维度评价

### 1. 统计严谨性：★★★★★（优秀）

这是项目最突出的优势。

- **拒绝 NHST 单一依赖**：不使用"p<0.05 即发表"的套路。每个对比同时报告 Cohen's d（效应量）、bootstrap/t 分布 CI（精度）、置换检验 p-value（显著性）——三线证据互相校验。
- **可复现性机制**：mulberry32 种子 PRNG 贯穿全部随机路径（LLM seed、发言者选择、shuffle 旋转、bootstrap 重采样、置换检验）。同等条件下结果可精确复现。
- **效应量优先**：文档中对"显著但效应小"和"不显著但效应大"做了区分（如 A vs B 异步对比：d=0.95、p=0.051——方向支持、效应大、功效不足），避免了 p 值的二分陷阱。
- **小样本校正**：CI 使用 t 分布而非正态近似（n=10 时差异可达 40%），置换检验使用 (count+1)/(nPerm+1) 保守估计。
- **错误隔离**：实验级 try-catch + 指数退避重试 + 错误占位文件，单个 API 故障不会污染统计数据。
- **诚实声明**：文档多处标注"未实验验证""启发式设定""功效不足"——这在独立项目中罕见。

**扣分项**：部分分析脚本（analyze.ts 等）仍使用 bootstrap CI 而非 t 分布 CI（小样本下 bootstrap 可能低估尾部）。Shuffle 旋转在修复前是固定的（已修复）。

### 2. 方法学原创性：★★★★☆（很强）

- **Shuffle 对照**（★★★★★）：这是项目最具原创性的方法论贡献。通过旋转 agent 独有知识打破角色-信息一致性，同时保留信息总量，精确测量了"信息整合的理论上限"。它不是 agent 协作的上限（shuffle 破坏了协作所需的角色连贯性），但标定了"如果信息完全流通能达到多好"的绝对参照系。这种方法学在 LLM multi-agent 领域是新的。

- **热力学终止隐喻**（★★★★☆）：将 Kuramoto 序参量 R + Shannon 熵 H + 归一化温度 T + 自由能 F = (1-R)+T·H 引入讨论终止决策。物理隐喻在社会科学中的移植通常危险，但此处的映射是合理的：R 测量相位同步（信念方向一致）、H 测量分布离散（信念 bin 分布）、T 测量数值方差。两轮阈值重新标定（硬截断 40%→10%）证明了这个框架是可操作的，不是纯粹的理论装饰。

- **F 分解驱动的干预优先级排序**（★★★☆☆）：当多个检测器同时触发时（91.7% 的场景），用 F = U + T·H 的分解项（结构性无序 vs 热性无序）驱动干预优先级。回测 97 次 force_reflection 事件发现原假设被证伪（p=0.041），但修正后的映射（thermal·(1-structural)）方向正确。这个思路有价值但仅在一个数据集上回测过。

- **D 组匹配分布设计**（★★★☆☆）：D 组从 C 组的实际终止分布中采样终止点，使得 C/D 对比的唯一差异是"终止决策质量"。这在实验设计上是干净的——控制了"讨论总量"这个混淆变量。n=10 功效不足是主要限制。

### 3. 核心发现：★★★★☆（强，但受限于验证范围）

| 发现 | 证据强度 | 新颖性 | 普适性 |
|------|---------|--------|--------|
| 虚假共识 r≈0 | ★★★★★ n=169, 跨 2 任务, p=0.66 | ★★★ 人类群体已知，LLM 群体首次量化 | ⚠️ 仅 DeepSeek-V3 + FlatTopology |
| 治理药物动力学 | ★★★★☆ p=0.015, 效应一致 | ★★★ 首次在 LLM 中量化治理时间动力学 | ⚠️ 仅 3 轮限制，更长讨论未测试 |
| Shuffle 信息上限 | ★★★★☆ d=1.82, p<0.001 | ★★★ 方法学新 | ⚠️ Supplier 任务天花板效应 |
| 治理有效 (Crisis) | ★★★★☆ d=0.92, p=0.005, 功效 88% | ★★☆ 方向已知，规模/严格性新 | ⚠️ 单模型 |
| H_thermo→阈值标定 | ★★★☆☆ 两轮迭代验证 | ★★☆ 工程贡献 | ⚠️ 仅 fraud 任务 |

"虚假共识"是最重要的发现——它直接证明了认知治理的必要性。如果共识天然导向正确，治理只需要促进信息共享；但 r≈0 意味着光促进共识不够，必须纠偏。这个发现的限制是仅在 DeepSeek-V3 上验证——GPT-4o 或 Claude 的一致性问题可能程度不同。

## 三、最突出的风险

### 风险 1：单模型依赖（严重程度：高）

445 次实验中 391 次使用 DeepSeek-V3，54 次跨模型验证（Zhipu 14 + Qwen 40）。虚假共识（r≈0）这一最重磅的发现在 DeepSeek-V3 上验证，但跨模型对比（Zhipu C 组 τ=0.76 > DeepSeek 0.64，+18.8%）已初步支持热力学终止方向的有效性。

**缓解方案**：至少跑 GPT-4o-mini 和 Claude Haiku 各一组（n=10, Crisis none/full/shuffle），成本约 ¥20-50。如果 r≈0 在三个模型上都复现，普适性主张将大幅增强。

### 风险 2：共识只在 FlatTopology 下测量（严重程度：中）

所有 agent 看到所有消息。在真实的分组/层级拓扑中，局部共识和全局共识的关系可能不同——子群内部可能形成高共识的"回音室"，使虚假共识问题更严重。

**缓解方案**：至少测试 GroupedTopology（3+2 分组讨论 → 代表汇报 → 全体讨论）。

### 风险 3：全部为排序任务（严重程度：中）

Kendall's τ 测量排名一致性。分类任务（如"哪个方案是正确的"）和估计任务（如"预测感染人数"）上，共识-质量关系是否仍为零是未知的。

### 风险 4：实现的模块远多于验证的模块（严重程度：低）

README 中列出的自适应阈值、自适应剂量、GroupedTopology、CommitteeTopology 等已实现但从未进入实验。这不是质量问题，但造成了文档和现实的差距。

## 四、这个项目最值得被记住的

如果只能选三样：

1. **虚假共识的量化**（r ≈ −0.05, n=169, 跨 2 任务）。"LLM 多智能体讨论中，最终共识度与最终正确率的线性相关 ≈ 0"——这是一个统计上健壮、方法学上干净的零发现。它为"认知治理"提供了存在理由。

2. **Shuffle 对照的方法学**。用信息旋转打破角色-信息一致性来测量信息整合的理论上限——这个方法可以被任何 LLM multi-agent 实验采用。

3. **诚实声明的文化**。在文档中明确标注"未实验验证""启发式设定""功效不足"——这在追求"看起来完美"的技术项目中极度稀缺，是科学精神的体现。
