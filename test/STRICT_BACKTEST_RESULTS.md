# SwarmAlpha 严格回测报告

> 无信息泄漏 · 全新事件 · 真实价格验证 · 14 事件多样化测试
>
> 2026-06-21 (v4.1)

---

## 方法

- **14 个历史事件**（8 up + 5 down + 1 neutral），**均不在**现有 17 事件训练数据库中
- 只用**事发当天已知的信息**（VIX/RSI 来自当日收盘，政策响应限于已公告内容）
- "正确答案"来自**真实后续 1-3 个月回报**，不是数据库字段
- 对比三个系统：纯校准、纯 LLM（模拟）、混合预测

---

## v4.0 → v4.1 修复内容

v4.0 的三个系统均低于随机（校准 0%、LLM 0%、混合 25%）。根因与修复：

| # | v4.0 问题 | v4.1 修复 | 效果 |
|---|----------|----------|------|
| 1 | 模拟 LLM 硬编码看空（起点 -40） | 中性起点 + RSI/VIX 反向信号 + 政策加分 | 0% → 35.7% |
| 2 | 校准跌幅惩罚 ×3 压制超卖奖励 | 降为 ×1.5 + 分级超卖奖励 + 恐慌极值检测 | 0% → 42.9% |
| 3 | 分类器覆盖权重太低 | V_REBOUND 权重升至 50-80% + RSI 直接加分 | 25% → 64.3% |
| 4 | 政策检测遗漏"协调/正在"等状态 | 识别进行中的政策行动 | LTCM 正确分类 |
| 5 | 低置信度分类器覆盖校准正确判断 | 置信度 < 40% + 矛盾 → 信任校准 | 平安夜暴跌正确 |

## 诚实结论

1. **混合预测 64.3%** — 较最佳单一系统（校准 42.9%）提升 +21pp。架构有效，但远非完美
2. **校准在 up 事件上出色（75%）但在 down 事件上完全失败（0%）** — 系统的超卖看多偏差在真正下跌时是毒药
3. **LLM 在 down 事件上表现最好（40%）** — 模拟 LLM 的看空倾向在熊市中有价值
4. **核心张力未解决**：超卖 = 买入（V 型反弹）vs 超卖 = 还有更多下跌（L 型衰退）。区分两者的能力仍不足

| 2015 瑞央行 | neutral | up | 一次性冲击被误判为V型反弹机会 |

### 64.3% 意味着什么

- ✅ **方向比抛硬币好 2 倍**（64% vs 33%）
- ✅ **混合架构确实比单一系统强**（+21pp）
- ⚠️ **在 up 事件上可靠（100%），在 down 事件上不可靠（20%）**
- ⚠️ **14 个事件仍太少，统计显著性不足**
- ⚠️ **需要真实 LLM（非模拟）验证**

## 改进方向

1. **区分 V vs L 的核心能力** — 当前分类器过于依赖 RSI/VIX/政策响应，需要加入杠杆强制平仓程度、信贷利差、市场结构数据
2. **真实 LLM 测试** — 模拟 LLM 只是近似。真实 DeepSeek 的 5-Agent 博弈会有更细致的判断
3. **更大样本（50+）** — 14 个事件不足以做统计推断。需要每个方向至少 20 个事件
4. **时间维度** — 目前只预测方向，没有预测反弹时点。2020 新冠案例中系统方向可能是对的（最终反弹了），但时点错了（预测反弹太早）
5. **真实市场数据** — 替换从新闻文本推断的 VIX/RSI/跌幅，使用真实历史数据
