## 二、非对称门控诊断

### 实验设计
在 78 事件上对比 `disableClustering=true`（纯线性共识）vs `false`（KMeans 门控开启），模板模式，Neutral OFF。

### 结果
```
❌ 门控关闭 (纯线性共识)    Total: 46.2%  Up=39%  Down=79%
✅ 门控开启 (KMeans<-15)    Total: 46.2%  Up=39%  Down=79%
```

**门控开关不影响最终准确率。完全相同。**

### 根因分析
从日志中提取门控决策统计：780 次共识决策中 **46% 使用了 cluster 共识**。但门控改的是共识**幅度**（如 -8.7 → -37.2），不改**方向**——因为两种共识方法在 `+15` 阈值的哪一侧总是达成一致。

**真正的瓶颈在 `+15` 方向判定阈值**，不是门控算法。

## 三、方向阈值扫描

### 模板模式 (78 事件)

| 阈值 | Total | Up | Down |
|------|-------|-----|------|
| +15 (默认) | 46.2% | 39% | 79% |
| +10 | 43.6% | 39% | 71% |
| +5 | 46.2% | 45% | 67% |
| 0 | 46.2% | 45% | 67% |
| **-5** | **52.6%** | **61%** | **58%** |
| -10 | 52.6% | 61% | 58% |

> 模板最优阈值 = -5，但仍然低于永远猜涨基线 56.4%

### LLM 模式 (20 事件, DeepSeek)

| 阈值 | Total | Up | Down |
|------|-------|-----|------|
| +15 (默认) | 55% | 70% | 57% |
| +10 | 55% | 70% | 57% |
| **+5** | **65%** | **90%** | **57%** |
| 0 | 65% | 90% | 57% |
| -5 | 65% | 90% | 57% |

> LLM 最优阈值 = +5，总准确率 65%，远超永远猜涨基线

### 核心发现

1. **LLM 因子质量碾压模板** — 65% vs 52.6%，差距 12.4pp
2. **模板模式天花板在因子提取**，不在共识算法
3. **LLM 模式对阈值不敏感**（+5~-5 区间内 Up 都 90%），因为因子信号本身够强
4. 永远猜涨基线只有模板模式下才有意义——LLM 模式轻松超越

## 四、引擎改动

### 新增 `directionThreshold` 配置

**改了 3 个文件**：

| 文件 | 改动 |
|------|------|
| `v9/types.ts` | `V9SimConfig` 新增 `directionThreshold?: number` |
| `v9/uncertaintyEngine.ts` | `NeutralInputs` 新增 `directionThreshold`；`makeDecision()` 三个分支全部使用可配置阈值 |
| `v9/simulation.ts` | 从 config 传递 `directionThreshold` 到 `makeDecision` |

**使用方式**：
```typescript
// 模板模式: 降低阈值
POST /api/swarm { "version": "v9", "news": "...", "directionThreshold": -5 }

// LLM 模式: 保持默认 15
POST /api/swarm { "version": "v9", "news": "...", "llmConfig": {...} }
```
