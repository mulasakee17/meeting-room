## 1. Introduction

### 1.1 问题：认知治理层的缺失

LLM 多 Agent 系统正从研究原型转向生产部署。AutoGen、CrewAI、LangGraph 等框架协调 AI Agent 团队完成金融分析、临床决策、法律推理。然而这些系统继承了人类群体的集体决策缺陷：回声室、权威偏误、群体极化、过早共识。

2026 年涌现的"Agent 治理"工具——Microsoft Agent Governance Toolkit、Agent Control Standard (ACS)、NVIDIA OpenShell——全部聚焦**安全层**（未授权工具调用、预算超支、数据泄露）。**认知层**（检测 Agent 讨论何时偏向集体失败）缺乏生产级工具。

SwarmAlpha 填补这一空白：一个可嵌入的认知治理运行时，观察、检测、干预多 Agent 系统的集体决策失败。

### 1.2 研究问题

本报告围绕三个递进的研究问题：

| # | 研究问题 | 状态 |
|---|---------|------|
| RQ1 | 认知治理干预是否提升多 Agent 决策质量？ | ✅ Crisis 任务显著有效（d=0.92, p=0.005） |
| RQ2 | 社会热力学 F 分解能否为干预优先级提供理论框架？ | ⚠️ 诊断价值已验证，运行时排序价值未验证 |
| RQ3 | F 分解驱动的干预排序相比固定排序是否提升决策质量？ | ❌ A/B 对照未支持（d_z=−0.354, p=0.378） |

### 1.3 核心贡献

1. **治理有效性实证**：修复 4 个认知缺陷（D1-D4）后，Crisis 任务上治理显著提升 τ（d=0.92, p=0.005, power=88%）
2. **社会热力学诊断框架**：F 分解提供了分析框架，发现并修正了 force_reflection 映射的错误假设（H1 证伪，p=0.0092，2026-07-21 修复 Round 3 零值 bug 后）
3. **诚实的负面发现**：F 分解排序的 A/B 对照实验未显示显著改善，记录了边界条件
4. **范式批判**：识别多 Agent 讨论范式的 4 个认知缺陷（D1-D4），揭示治理环路断裂问题

## 2. 实验发现

### 2.1 实验设计

**2×2 因子设计**，两个任务 × 多个消融条件：

| 任务 | 难度 | 特征 | 样本量（闭环） |
|------|------|------|--------------|
| **Crisis Response** | 困难 | 5 个专业角色，隐藏信息需交叉验证 | 72 实验 |
| **Supplier Selection** | 简单 | 5 个采购角色，信息较透明 | 89 实验 |

**消融条件**：
- `none`：无治理基线
- `full`：完整治理（4 检测器 + F 分解排序 + 全部干预）
- `full_fixed`：完整治理但固定排序（A/B 对照 B 组）
- `shuffle`：打乱角色-信息绑定（结构干预，非认知干预）
- `full_diversity` / `full_weight` / `full_reflection` / `full_continue`：单干预消融

**评估指标**：Kendall's τ-b（agent 集体排序与 ground truth 排序的等级相关，tie 修正）

### 2.2 核心结果：治理有效性

| 任务 | 基线 τ (none) | 治理 τ (full) | Δτ | Cohen's d | p-value | Power |
|------|-------------|-------------|-----|-----------|---------|-------|
| **Crisis** | 0.41 | 0.62 | +0.21 | **0.92** | **0.005** | 88% |
| **Supplier** | 0.68 | 0.73 | +0.05 | 0.47 | 0.089 | 43% |

**Crisis 任务**：治理显著有效。τ 提升 51%，大效应量（d=0.92），统计功效充足（88%）。

**Supplier 任务**：方向一致但未达显著。天花板效应——基线 τ=0.68 已接近 full τ=0.767，治理空间有限。

### 2.3 Shuffle 边界条件

Shuffle 干预（确定性 +2 轮转 agent 已知信息，角色标签固定）揭示了一个关键边界条件：

| 任务 | 基线 τ | Shuffle τ | Δτ | Cohen's d | p-value |
|------|--------|----------|-----|-----------|---------|
| **Crisis**（困难） | 0.41 | 0.72 | +0.31 | **1.44** | <0.001 |
| **Supplier**（简单） | 0.68 | 0.69 | +0.01 | 0.09 | 0.78 |

**关键发现**：打破角色-信息绑定在困难任务上产生巨大改善（d=1.44），但在简单任务上无效。原因是天花板效应——Supplier 基线已接近最优，结构干预无额外空间。

**理论含义**：多 Agent 共识的相变边界不仅由交互参数（温度、从众压力）决定，还由角色-信息分配的拓扑结构根本塑造——这是当前统计物理模型缺失的维度。

### 2.4 共识与正确性的解耦

Kendall's τ（共识度）与决策质量的相关性：r ≈ −0.14（两个任务均如此）。

**含义**：共识水平不是正确性的可靠代理——高共识可能是"集体一致地错"（虚假共识）。这构成了 F 分解框架中 `introduce_diversity` 干预的理论基础，同时也暴露了 F 分解的根本盲区（见 §3.3）。

### 2.5 历史对照：165 环路断裂实验

## 3. F 分解框架

### 3.1 理论基础：社会热力学

将多 Agent 系统类比为热力学系统，提供 3 个状态变量：

| 变量 | 物理含义 | 系统含义 | 计算来源 | 范围 |
|------|---------|---------|---------|------|
| **R**（序参量） | Kuramoto 同步度 | agent 信念方向同步程度 | `computeKuramotoOrder` | [0,1] |
| **T**（温度） | 热运动幅度 | 信念分散程度（标准差） | `normalizeTemperature(computeStd)` | [0,1] |
| **H**（熵） | 信息无序度 | 信念分布的不确定性 | `shannonEntropy`（5 bins, 归一化） | [0,1] |

**Helmholtz 自由能**：

$$F = U - TS = (1-R) + T \cdot H$$

- **(1−R)**：结构性无序——agent 信念方向未对齐
- **T·H**：热性无序——agent 信念既分散又分布广

这两个分量**正交**：可以同时高（既极化又噪声大）、同时低（真共识）、或一高一低（伪共识/伪极化）。

### 3.2 Kuramoto 序参量的关键修复（H4）

**原缺陷**：相位映射 `θ = π·b` 使 b=±0.99 都落在单位圆左侧附近，R≈1，将极化误判为共识。

**修复**：改为 `θ = (π/2)·b`：
- b=+1（强支持）→ θ=+π/2（单位圆顶部）
- b=−1（强反对）→ θ=−π/2（单位圆底部）
- 两者正对，R≈0（正确反映极化）

代码位置：[src/lib/governance/index.ts:532](./src/lib/governance/index.ts#L532)

### 3.3 干预-无序映射：四个可证伪假设

F 分解的核心设计：每种干预类型对应一种"无序病机"，F 分解告诉我们当前系统主要患哪种病。

| 干预类型 | 评分公式 | 设计假设（可证伪） | 回测状态 |
|---------|---------|-------------------|---------|
| `force_reflection` | `thermal·(1−structural)`（原 `structural`，H1 证伪后修正） | 原假设：反思对齐方向。**证伪**后修正为：反思是降噪干预 | ✅ 证伪→修正 |
| `reduce_weight` | `thermal`（即 T·H） | 假设权重压制抑制高噪 agent | ⚠️ 方向支持不显著 |
| `introduce_diversity` | `R·(1−H)` | 假设"高同步低熵"= 虚假共识 | ⏳ 未回测 |
| `continue_discussion` | `R·(1−H)·(1−F)` | 假设有序+低F+早期=过早收敛 | ❌ 证伪+禁用 |

**关键诚实声明**：这 4 个公式是基于干预机制的作用假设**设计**的，不是从理论**推导**的。每个公式背后是一个**可证伪的假设**，而非数学必然。

### 3.4 根本盲区：F 分解无法区分真/假共识

`R·(1−H)` 对"真共识"和"虚假共识"给出相同评分——两者都是高 R 低 H。区分需要 ground truth 或外部信号，F 分解本身做不到。这是三层架构中层 3（增量评估反馈）要解决的核心问题。

### 3.5 三层联合架构

| 层 | 功能 | 状态 |
|----|------|------|
| **层 1** | F 分解驱动干预优先级排序 | ✅ 已实现 |
| **层 2** | 任务难度感知门控（τ₁ + ΔF 轨迹调剂量） | 🔄 实验室 |
| **层 3** | 在线干预效果反馈（ΔT 自动降权有害干预） | 🔄 实验室 |

**层 2 原设计推翻**：原设计"F 低→收紧阈值"在天花板情形下会触发更多干预，加剧有害干预。推翻后改为任务难度感知门控：τ₁ 高 → dosage_scale 低 → 抑制干预。

## 4. 证伪与修正

### 4.1 回测方法论

**数据**：161 闭环实验中每次 force_reflection / reduce_weight 干预事件。

**方法**：按干预时系统的 (1−R) vs T·H 比值分桶，对比结构性主导组 vs 热性主导组的 Δτ（下一轮 tau − 当前轮 tau）。

**统计**：配对置换检验（5000 perms，(count+1)/(nPerms+1) 修正）+ Cohen's d。

**局限声明**：观察性研究，非因果确证（agent 在不同 F-state 非随机分配，存在混杂）。

### 4.2 假设 1（force_reflection ↔ structural）：证伪并修正

**原假设**：force_reflection 是对齐方向的干预，应在结构性无序主导时有效。

**回测结果**（85 事件，排除 Round 3 终止事件；2026-07-21 修复零值 bug）：

| 组 | n | 平均 Δτ |
|----|---|--------|
| 结构性主导 (1−R > T·H) | 23 | **−0.035**（有害） |
| 热性主导 (T·H ≥ 1−R) | 62 | **+0.184**（有益） |

- 置换检验 p = 0.0092（5000 perms）
- Cohen's d = −0.667
- 方向差（热性 − 结构性）= +0.219

**结论**：**假设 1 证伪**。force_reflection 在结构性主导（极化）时有害，在热性主导时有益。force_reflection 不是对齐方向干预，而是降噪干预。

**修复记录（2026-07-21）**：原回测脚本 `backtest_weight_assumption.ts:114-115` 对最后一轮（Round 3）事件强制 `Δτ=0`（因无下一轮 tau），导致热性组均值被零值拉低（Round 3 事件在热性组中占比更高）。修复后改为跳过无下一轮的事件，p 从 0.041 降至 0.0092，效应量从 d=−0.49 扩大至 d=−0.667。H1 证伪方向稳健且更强。

**修正**：评分公式从 `structural` 改为 `thermal·(1−structural)`——在热性主导且非极化时优先。

代码位置：[src/lib/governance/index.ts:567](./src/lib/governance/index.ts#L567)

### 4.3 假设 2（reduce_weight ↔ thermal）：方向支持且显著

**原假设**：reduce_weight 是热性无序干预，应在热性主导时有效。

**回测结果**（81 事件，排除 Round 3；2026-07-21 修复）：

| 组 | n | 平均 Δτ |
|----|---|--------|
| 热性主导 | 64 | +0.247 |
| 结构性主导 | 17 | +0.071 |

- 置换检验 p = 0.023（5000 perms）
- Cohen's d = +0.662

**结论**：**假设 2 方向确认且统计显著**。热性主导时 reduce_weight 的 Δτ 显著高于结构性主导，支持 reduce_weight 作为热性无序干预的设计假设。

### 4.4 假设 3（introduce_diversity ↔ 虚假共识）：未回测

**未回测原因**：echo chamber 检测器在 mock 测试中难以触发，需真实 LLM 消息数据。

**根本盲区**：即使回测，F 分解也无法区分"真共识"与"虚假共识"——两者都是高 R 低 H。这是层 3 要解决的问题。

### 4.5 假设 4（continue_discussion ↔ 过早收敛）：证伪并禁用

**实验结果**：Crisis 任务上 0% 有效率，平均 Δτ = −0.400（有害）。

**结论**：**假设 4 证伪**。continue_discussion 已默认禁用（`disabledInterventions`）。

### 4.6 证伪的价值

4 个假设中，1 个证伪并修正，1 个方向支持但不显著，1 个未回测，1 个证伪并禁用。这个证伪过程本身是 F 分解框架的核心价值——它提供了**结构化的、可证伪的分析框架**，使假设错误能被发现和修正。

## 5. A/B 对照实验

### 5.1 实验动机

§4 的回测是观察性研究，无法孤立 F 分解排序对决策质量的因果贡献。为回答"F 分解排序是否比固定排序更好"，设计了预注册的 A/B 配对对照实验。

### 5.2 实验设计

**预注册假设 H_F**：F 分解排序的 Δτ 显著高于固定排序（单尾）。

**配对设计**：
- A 组：现有 `crisis_full` 实验（F 分解排序，sortingMode='fdecomposition'）
- B 组：新增 `crisis_full_fixed` 实验（固定排序，sortingMode='fixed'）
- 配对：同 runIndex（相同 seed），A/B 仅在排序逻辑上不同

**固定排序定义**：保持检测器触发顺序（reduce_weight → introduce_diversity → force_reflection → continue_discussion），不按系统状态重排。代表"未引入 F 分解前"的现实基线。

**统计方法**：
- 主检验：配对置换检验（符号翻转，10000 perms，(count+1)/(nPerms+1) 修正）
- 效应量：Cohen's d_z = mean(Δτ_A − Δτ_B) / std(Δτ_A − Δτ_B)
- CI：t 分布配对均值差的 95% CI
- 双分析：ITT（所有配对）+ per-protocol（仅排序实际改变的配对）

**Pilot 策略**：先跑 n=8 配对，根据 d_z 决定是否扩样。

### 5.3 Pilot 结果（n=8，Crisis 任务）

| 指标 | ITT（全部 8 配对） | Per-protocol（8/8 排序改变） |
|------|-------------------|---------------------------|
| A 组 τ 均值（F 分解） | 0.6250 | 0.6250 |
| B 组 τ 均值（固定排序） | 0.6750 | 0.6750 |
| 配对差 Δτ_A − Δτ_B | **−0.0500** | **−0.0500** |
| Cohen's d_z | **−0.354** | **−0.354** |
| 配对置换检验 p-value | 0.3781 | 0.3765 |
| 95% CI | [−0.168, +0.068] | [−0.168, +0.068] |

**配对明细**：

| runIndex | τ_A (F 分解) | τ_B (固定) | diff | 排序不同 |
|----------|-------------|-----------|------|---------|
| 0 | 0.600 | 0.600 | 0.000 | 是 |
| 1 | 1.000 | 1.000 | 0.000 | 是 |
| 2 | 0.200 | 0.400 | −0.200 | 是 |
| 3 | 0.800 | 1.000 | −0.200 | 是 |
| 4 | 1.000 | 0.800 | +0.200 | 是 |
| 5 | 0.600 | 0.600 | 0.000 | 是 |
| 6 | 0.400 | 0.400 | 0.000 | 是 |
| 7 | 0.400 | 0.600 | −0.200 | 是 |

### 5.4 结论：H_F 不支持

**H_F 不支持**。8/8 配对中 F 分解确实改变了排序顺序（per-protocol 排除率 0%），但**改变排序没有改善 τ**，方向上反而略差（d_z=−0.354）。

- 4/8 配对 τ 相同（排序改变但结果不变）
- 3/8 固定排序更优（diff=−0.2）
- 1/8 F 分解更优（diff=+0.2）

**停止扩样决策**：基于预注册决策规则（d_z<0.2 或方向反转则停止），pilot 已显示方向反转，扩样无法翻转结论。

### 5.5 为什么未改善——三个可能解释

1. **3 轮太短**：同轮内多个干预都会执行（只是顺序不同），排序对最终 τ 影响有限——解释了 4/8 配对 τ 完全相同

2. **H1 修正后与固定排序重合**：force_reflection 降权后，F 分解和固定排序在 Crisis 上都把 reduce_weight 排前面，效果重合

3. **固定排序恰好接近最优**：Crisis 上 authority bias 和 polarization 最常见，固定排序 reduce_weight 优先恰好符合热性主导场景

### 5.6 F 分解价值的修正定性

| 维度 | F 分解贡献 | 证据强度 |
|------|-----------|---------|
| 诊断价值（发现 H1 错误） | ✅ 实质贡献 | 强（p=0.0092, d=−0.667） |
| 统一多检测器优先级（架构） | ✅ 架构合理 | 弱（无 Δτ 改善证据） |
| 提升决策质量（Δτ） | ❌ 未验证 | A/B 对照 d_z=−0.354，方向反转 |

**结论**：F 分解的主要价值是**诊断性**的——它提供了分析框架帮助发现 force_reflection 映射的错误假设。作为运行时干预排序机制，在 Crisis 任务 3 轮讨论中相比固定排序没有显著改善。更长讨论轮次或多任务场景下是否有效，留实验室验证。

## 6. 范式批判

### 6.1 多 Agent 讨论范式的四个认知缺陷（D1-D4）

在实验过程中，诊断出多 Agent 讨论范式存在 4 个根本认知缺陷，导致治理环路断裂：

| # | 认知缺陷 | 症状 | 影响 | 修复 |
|---|---------|------|------|------|
| **D1** | **状态感知缺失** | `buildPrompt` 未注入 agent 当前 belief/confidence，状态修改干预对 LLM 不可见 | reduce_weight、force_reflection 等干预无效 | buildPrompt 注入当前 belief/confidence |
| **D2** | **无对话历史** | 所有 agent 看同一全局摘要，不知道自己之前说过什么 | agent 无法维持立场一致性 | 个性化记忆：注入自己的历史发言和 @提及 |
| **D3** | **同步轮次（并行"念稿"）** | `Promise.all` 让所有 agent 并行发言 | agent 看不到同轮已发言的同伴 | 改为顺序 for 循环 |
| **D4** | **虚构影响力网络** | 同意/反对/说服边从信念数值差异**推断** | 产生虚假影响力幻影网络 | 仅从显式 referencedAgents 构建边 |

### 6.2 治理环路断裂的后果

D1-D4 导致 165 个历史实验中，状态修改干预**从未到达 agent 感知**：
- D1 使 reduce_weight/force_reflection 不可见（agent 感觉不到权重变化）
- D2 使 agent 无法跨轮维持立场（每轮"失忆"重新发言）
- D3 使同轮 agent 互不可见（并行念稿而非真正讨论）
- D4 使影响力图谱是虚构的（基于数值差而非真实引用）

**后果**：165 历史实验的"治理无效"结论是**环路断裂的假象**，不是治理本身无效。修复 D1-D4 后的 161 闭环实验显示 Crisis 任务治理显著有效（d=0.92）。

### 6.3 对范式的批判性思考

**当前多 Agent 讨论范式的根本问题**：多数框架假设"让 agent 交流就能涌现好的集体决策"，但忽略了：

1. **状态可见性**：agent 必须能感知自身状态变化，否则干预无效
2. **个体记忆**：agent 必须知道自己说过什么，否则无法维持立场
3. **顺序交互**：真正的讨论需要顺序发言，并行"念稿"不是讨论
4. **真实引用**：影响力网络必须基于显式引用，数值推断会产生幻影

**启示**：多 Agent 系统的"涌现"质量取决于底层交互机制的正确性。在断裂的范式上做实验，结论是不可信的——这正是 165 历史实验的教训。

### 6.4 统计方法的诚信修正

在实验过程中发现并修正了多个统计方法问题：

| 问题 | 原做法 | 修正 |
|------|--------|------|
| p=0.000 假阳性 | 置换检验无修正 | (count+1)/(nPerms+1) 修正 |
| τ-a 忽略 ties | 使用 τ-a | 改用 τ-b（tie 修正） |
| t 分布表不完整 | 线性插值 | 补全 1-30 整数值 |
| BH FDR 非标准 | 简单阈值 | 标准步降程序 |
| Welch p 值小样本偏差 | 无修正 | t_critical/z_critical 修正因子 |

### 7.2 理论方向

1. **F 分解的诊断价值深化**：F 分解作为分析框架已证明有效（发现 H1 错误），但作为运行时排序机制未验证。需要理论分析为何在 3 轮讨论中排序影响有限——可能是"同轮干预集合相同时，顺序对最终收敛轨迹的二阶效应"。

2. **真/假共识区分**：F 分解无法区分真共识与虚假共识。层 3 的在线 Δτ 反馈是一种经验性学习方案，但理论上的 ground truth-free 区分方法仍需探索。

3. **角色-信息拓扑**：shuffle 实验（d=1.44）显示角色-信息绑定是相变边界的结构前兆。这指向一个新理论方向：多 Agent 共识的质量不仅由交互动力学决定，还由初始信息分配的拓扑结构根本塑造。

### 7.3 方法学方向

1. **因果推断强化**：当前回测是观察性研究。理想方案是随机化干预排序（A/B 对照已部分实现），但完整因果推断需要更精细的实验设计。

2. **跨任务泛化验证**：当前结论基于 Crisis + Supplier 两任务。需要更多任务类型（如弱依赖任务、强依赖任务）验证治理有效性的边界条件。

3. **成本-效益分析**：CustomAgent 已追踪 token 用量、API 调用数、延迟。需要系统分析治理的边际成本与边际收益。

## 8. 2026-07-20 结论审计与修正

> 本节记录对原结论的审计结果，包括 C1 撤回、新发现 F8-F10、Supplier 天花板部分证伪、理论命题修正。

### 8.1 C1 结论撤回（force_reflection "反向强化 +0.68" 归因错误）

**原结论 C1**（来自 E 组恶意 agent 实验）：force_reflection 使恶意 a1 信念平均 +0.68，表现为"反向强化"。

**审计发现**：100% 的 force_reflection 样本（5/5）同轮都有 reduce_weight 干预打诚实 agent，**归因完全混淆**。原结论无法区分是 force_reflection 单独作用还是 reduce_weight 间接影响。

**修正定性**：
- 原结论 C1（+0.68）**撤回**
- 部分隔离分析（force_reflection 打 a1，reduce_weight 打别人）：n=5，5/5 上升，平均 +0.94（F9 发现）
- 但 n=5 极小，且"同轮 reduce_weight 打诚实 agent"可能间接影响 a1，**仍非严格因果隔离**

**教训**：观察性研究中多重干预同轮发生时，归因必须严格检查混淆变量。原 C1 结论是"确认偏误"的典型案例——看到预期模式（force_reflection 后 a1 上升）就归因，忽略了同轮其他干预。

### 8.2 新发现 F8-F10（E 组深度分析）

基于 E 组 n=10 v2 trace 数据的深度分析（`experiments/v2/analyze_e_depth.ts`）产生 3 个新发现：

| # | 发现 | 数据 | 反常识程度 |
|---|---|---|---|
| **F8** | 失败组 token 成本是成功组的 2.9 倍（96K vs 33K） | n=6（成功 4 + 失败 2） | ★★★☆☆ |
| **F9** | force_reflection 部分隔离下 5/5 反向强化（平均 +0.94） | n=5 | ★★★★☆ |
| **F10** | reduce_weight 部分隔离下 72% 压制率（平均 Δa1=-0.13） | n=18 | ★★☆☆☆ |

**F8 解读**：常识预期"治理失败 = 干预无效 = 浪费的 token 少"。实际数据是"治理失败 = 干预反复 = 消耗 2.9 倍 token"。失败组（如 run#3）干预 14 次、轮次 17 轮，反复拉锯导致 token 暴涨。

**F9/F10 对比**：在部分隔离观察下，reduce_weight 有效压制（72%），force_reflection 反向强化（100%）。这比"归因不清"更强，但仍非严格因果隔离。

详细数据见本文档附录 B。

### 8.3 Supplier 天花板效应部分证伪

**原结论**：Supplier 任务 shuffle 无效应（d=0.09）因天花板效应——基线 τ=0.68 已接近最优。

**证伪数据**：none 组 30 次实验中，**47% (14/30) 的 τ < 0.8**，说明治理空间存在。

**修正定性**：
- "天花板效应"可能部分是**功效不足**（n=30，43% power）而非真实天花板
- 需扩样至 n=72 达 80% 功效才能定性
- 当前 p=0.089 可能是 power 问题而非 true null

### 8.4 THEORY.md 命题 1b/1c/2 修正

经 `experiments/v2/test_theory_propositions.ts` 脚本测试，THEORY.md 中 3 个命题表述错误：

| 命题 | 原表述 | 修正后 |
|---|---|---|
| 1b | 完美两极分化 → R=0 | **仅偶数 N 且完美对半分时 R=0**；奇数 N 或含中间值时 R>0 |
| 1c | 均匀分布 → R≈2/π | **仅 N→∞ 连续极限时 R→2/π**；有限 N 下偏离 |
| 2 | R-H 互补（隐含严格阈值） | **定性趋势**，非严格阈值 |

**影响**：THEORY.md 命题 1a、3 仍成立。理论分析整体方向正确，但部分细节（阈值、连续性假设）需更严谨。

### 8.5 审计的方法学价值

本次审计的核心价值不是"发现错误"，而是**证明了项目自我纠错机制的有效性**：

1. **可证伪性设计**：所有结论都预设了证伪条件（见 [LIMITATIONS.md 附录 A](./LIMITATIONS.md)），使错误能被发现
2. **归因审计**：对每个结论检查混淆变量，发现 C1 的归因错误
3. **理论测试**：用脚本测试理论命题，发现表述错误
4. **边界条件复核**：对"天花板效应"等边界结论用原始数据复核

**这本身就是治理系统价值的元证据**：一个能检测自身错误的系统，比一个声称永远正确的系统更可信。

## 二、设计缺陷复查

> **2026-07-19 更新**：缺陷 #1、#2、#3 已修复。新增 §五（本轮审计新发现）。

### 🔴 严重缺陷

#### 缺陷 1：extractRanking 的两种提取路径不一致 ✅ 已修复（2026-07-18）

**修复**：删除了 V1 fallback（首次提及位置启发式），`extractRanking` 现在仅使用 itemBeliefs 聚合路径。itemBeliefs 为空时抛出错误（调用方已有 try-catch 隔离）。

**注**：`run.ts` 和 `run_async_ab.ts`（主实验链路）已迁移到 V2。`sensitivity.ts` 仍用 V1 indexOf 路径——该脚本的 agent prompt 仅要求输出 `{emotion, reasoning}`，不要求 itemBeliefs 字段，迁移需改 prompt 会使现有敏感性扫描数据失效。`sensitivity.ts` 的 `kendallTau` tie 修正公式 BUG 已于 2026-07-20 修复（与 run.ts 一致）。

#### 缺陷 2：consensusLevel 操作化与热力学引擎不一致 ✅ 已修复（2026-07-18）

**修复**：`run.ts` 的 `consensusLevel` 已替换为真实 `kuramotoR(beliefs)`（`θ = b × π/2`），与 `asyncEngine.ts` 中的 R 完全一致。

#### 缺陷 3：shuffle 旋转确定性（+2 固定偏移） ✅ 已修复（2026-07-18）

**修复**：`shuffleTask` 使用 `mulberry32(42 + runIndex)` 为每次实验生成随机旋转偏移量，不再固定 +2。

---

### 🟡 中等缺陷

#### 缺陷 4：单模型 + 单温度

406 次实验使用 DeepSeek-V3 + temperature=0.2，10 次跨模型验证使用 Zhipu glm-4-flash。仍缺：
- GPT-4o / Claude 等主流模型跨模型验证（仅 Zhipu 完成C 组）
- 温度敏感性（T=0 确定性和 T=1 创造性下的治理效果可能完全不同）
- 模型规模梯度（小模型 vs 大模型的一致性问题可能更严重）

**影响**：虚假共识（r≈0）发现主要基于 DeepSeek-V3，但 Zhipu C 组 τ=0.76 > DeepSeek 0.64（+18.8%）已初步支持热力学终止方向跨模型一致。

#### 缺陷 5：纯被试间设计——无法控制组间方差

每个实验是独立的。同一个随机种子下，`none` 和 `full` 的实验使用不同 seed（`seed = 42 + runIndex`），但条件间无配对。组间方差（有些 seed 恰好产生好/差的讨论）可能掩盖或放大治理效果。

**已有缓解**：大样本（n=24-30）部分平滑了组间方差。但配对设计（同一 seed 下 none 和 full 跑相同初始条件）会更敏感。

#### 缺陷 6：干预有效性阈值 0.05 无标定

```typescript
effect.effective = delta > 0.05; // 任意阈值
```

agent 信念的自然波动幅度未知。0.05 可能过高（错过真实但微小的干预效果）或过低（将随机噪声误判为有效）。

#### 缺陷 7：task_supplier 天花板效应

Supplier `none` τ=0.68。满分 τ=1.0，但随机排序 τ≈0。留给人机协作改进的空间只有 0.32。0.68 到 0.77 的提升（full τ=0.767）Δτ=+0.087，d=0.47——效应中等但天花板压制了改进幅度。

**影响**：Supplier 任务无法充分测试治理的上限——即使完美治理也无法超过 τ=1.0，但 0.68 的基线已经很高。适合用于验证"治理不有害"而非"治理大幅改进"。

---

### 🟢 轻微缺陷

#### 缺陷 8：仅 FlatTopology 验证

全部实验使用圆桌讨论（所有 agent 看到所有消息）。已实现但未实验验证的拓扑：GroupedTopology（分组讨论 + 代表汇报）、CommitteeTopology（委员会分层）。虚假共识（r≈0）可能在分组拓扑中更严重（组内回音室效应）。

#### 缺陷 9：仅排序任务

全部任务使用 Kendall's τ 排名。未测试：分类（多选一）、估计（连续值预测）、生成（开放文本质量）。共识-质量关系可能因任务类型而异。

#### 缺陷 10：3 轮限制

PARAMS.maxRounds=3。选择 3 轮的理由合理（治理干预需要多轮才能生效，第 1 轮信息分享、第 2 轮辩论、第 3 轮收敛），但异步引擎实验显示 28 次发言才收敛——同步 3 轮（15 次发言）可能过早截断。

#### 缺陷 11：无"个体判断"基线

没有"agent 独立判断、不讨论"的对照条件。这无法回答"讨论本身是否有用"——只能回答"治理讨论 vs 无治理讨论"。shuffle 部分回答了"信息整合的理论上限"，但不回答"讨论 vs 不讨论"。

#### 缺陷 12：Kendall's τ 仅测量排序一致性

τ 测量的是排名顺序匹配，但不测量：
- 排名之间的"距离"（将第 1 名错排为第 5 名 vs 第 2 名的惩罚相同）
- 排名的置信度校准
- 部分正确（前 2 名对但后 3 名错）

对于某些应用，Weighted τ 或 NDCG 可能更有信息量。

## 三、实验设计的优势（不应忽视）

1. **统计严谨性**：置换检验 + Cohen's d + bootstrap CI 的组合避免了参数假设。mulberry32 种子 PRNG 保证可复现。
2. **shuffle 对照的方法学创新**：通过打破角色-信息一致性同时保留信息总量，分离了"信息整合"和"讨论机制"的贡献。这是此项目最有原创性的方法论贡献。
3. **跨任务验证**：Crisis + Supplier 双任务独立验证，场景和基线难度不同——增强了发现的普适性。
4. **错误隔离**：run.ts 和 run_async_ab.ts 都有 try-catch + 重试 + 错误占位文件。单个实验失败不会中止整批。
5. **干预效果验证**：不仅记录"是否干预"，还测量干预后实际信念变化 > 0.05——区分了名义干预和实际有效干预。
6. **热力学轨迹完整保存**：async 实验保存了每 2 次发言的 R/T/H/F 完整快照，支持事后深度分析（如阈值标定的逐例尸检）。
7. **callLLM 重试**：providers.ts 的 3 次指数退避重试覆盖全部实验链路。

## 五、2026-07-19 本轮审计新发现

### 🔴 新发现 1：B 组和 D 组数据为 beliefShift 修复前生成

| 组别 | 时间戳 | beliefShift 状态 | 影响 |
|------|--------|-----------------|------|
| A | 2026-07-15 | N/A（sync DiscussionEngine） | 无影响 |
| B | 2026-07-15 | ❌ beliefShift=0 始终 | B vs C 对比被污染 |
| C (当前) | 2026-07-18 | ✅ 已修复 | 唯一干净数据 |
| D | 2026-07-16 | ❌ beliefShift=0 始终 | C vs D 对比被污染 |

**后果**：
- **B vs C 对比被污染**：C 组比 B 组多了一个有效的 beliefShift 信号，无法判断 C > B 是因为热力学终止还是 beliefShift。
- **C vs D 对比被污染**：C 有 beliefShift，D 没有，无法单独归因于"热力学 vs 随机"。
- **唯一干净的对比是 C 组三阶段自身**（旧阈值→新阈值→beliefShift 修复），三个阶段使用相同 K=3 和相同代码路径。

**修复（2026-07-19）**：B 组已重跑（含 beliefShift + K=2），D 组重跑中。新数据带 `codeVersion: "2026-07-19"` 标记。

### 🔴 新发现 2：evalEveryKUtterances 硬编码覆盖默认值

`run_async_ab.ts:328` 硬编码了 `evalEveryKUtterances: 3`，覆盖 `DEFAULT_ASYNC_CONFIG` 的默认值 2。**C 组三阶段所有实验（τ=0.34→0.46→0.64）实际都使用 K=3。** 文档声称的"K 从 3→2 提高了评估密度"从未在实验中生效——实验效果的提升完全来自 H/T/连续次数三个阈值改变 + beliefShift 修复，与 K 无关。

**修复（2026-07-19）**：`run_async_ab.ts:328` 改为 `evalEveryKUtterances: 2`。

### 🔴 新发现 3：A vs B 并非对等比较

A 组使用 `DiscussionEngine`（同步引擎），B 组使用 `AsyncDiscussionEngine`（异步引擎 `fixed_rounds=5`）。两者有不同的信念更新逻辑（异步引擎有 `updateListenerBeliefs`，同步引擎没有），且发言量不对等（B 组 5 eval 周期 ≈ 10-14 次发言 vs A 组 5 轮 = 25 次发言）。这不是"同步 vs 异步"的纯粹对比——两个引擎的代码路径完全不同。

### 🟡 新发现 4：D 组匹配分布基于旧 C 组数据

D 组从 C 组分布采样终止发言数（`sampleDTerminationPoint`）。旧 C 组平均发言 22.4-28.2，beliefShift 修复后的新 C 组平均约 18.6。D 组采样的分布已过时。D 组重跑时会自动使用新 C 组分布。

### 🟡 新发现 5：computeThermoState 温度用总体方差

`asyncEngine.ts:732` 的 `std` 计算除以 N（总体方差）而非 N-1（样本方差）。对 N=5 个 agent，总体方差比样本方差低估约 10%。T 的绝对值被系统性低估 → 结晶态检测更宽松。

**状态（2026-07-20）**：保持为设计选择。阈值 crystallT=0.22 等基于总体方差标定，改用 N-1 会使 T 增大 ~11.8%，需重新标定阈值 + 重跑全部异步实验（80+ 次）。代码中已添加注释说明此设计选择。

### 🟢 新发现 6：实验 JSON 缺少代码版本标记

已有实验数据无法从文件自身区分修复前后——只能通过时间戳推断。**修复（2026-07-19）**：`run.ts` 和 `run_async_ab.ts` 的结果对象已增加 `codeVersion: "2026-07-19"` 字段。

### 🟢 新发现 7：其他小问题

- `recentlySpoke` 惩罚只检查紧邻上一周期（可被"1-2-沉默-3-4"模式规避）
- `consensusDeviation` 使用全体均值，但被动倾听 agent 的信念更新滞后（学习率 0.15）
- 非发言者的 prompt token 消耗未被追踪（不影响 τ 但影响成本分析）
- `mulberry32` / `cohensD` / `mean` / `stdDev` 在 3+ 个文件中重复定义

## 1. 审核发现与修正记录

### 1.1 致命错误：Half-Normal 先验导致循环论证

**初始版本（错误）** 使用 Half-Normal 先验 `d ~ Half-N(0.5)`，报告 P(d>0)=100%。

**问题**：Half-Normal 先验在 d<0 时密度为 0，即先验已假设 d≥0。
- 先验 P(d>0) = 1
- 后验 P(d>0) = 1（必然）
- 这个 100% 完全来自先验假设，与数据无关——**循环论证**

**修正**：改为对称正态先验 `d ~ N(0, scale)`，允许 d<0。
- 先验 P(d>0) = 0.5（中性）
- 后验 P(d>0) 取决于数据——这才是真正的贝叶斯假设检验

### 1.2 其他修正

| 问题 | 修正 |
|------|------|
| 3 个假设检验无多重比较校正 | 加入 Bonferroni 校正报告 |
| 似然函数为正态近似未说明 | 加入方法学局限说明 |
| 先验名称从 "Half-N" 更新为 "N(0, scale)" | 全局替换 |

## 2. 修正后的诚实结论

### 2.1 核心结果

| 假设 | 频率派 p | Bonferroni 校正 p | 贝叶斯 P(d>0) N(0,0.5) | 95% HDI |
|------|---------|-------------------|----------------------|---------|
| Full vs None (τ) | 0.0365 * | 0.1095 | **96.7%** ** | [-0.04, 1.13] |
| Shuffle vs None | 0.0002 * | 0.0006 * | **100.0%** ** | [0.60, 1.72] |
| Full vs None (Q) | 0.0407 * | 0.1221 | **96.7%** ** | [-0.04, 1.13] |

`* p<0.05 频率派显著    ** P(d>0)>95% 贝叶斯确证`
`Bonferroni 校正 α = 0.05/3 = 0.0167`

### 2.2 诚实解读

**Full vs None（治理效应）：**
- 频率派未校正 p=0.037 显著，但 Bonferroni 校正后 p=0.110 不显著
- 贝叶斯 P(d>0)=96.7% 超过 95% 确证阈值——即使校正多重比较后，贝叶斯仍支持治理有效
- **但**：95% HDI 下界为 -0.03，略低于 0，说明仍有微小的不确定性
- **结论**：治理有效的后验概率 96.7%，但未达"确定性确证"（HDI 完全不含 0）

**Shuffle vs None（信息整合上限）：**
- 频率派 p=0.0002 极显著，Bonferroni 校正后仍显著
- 贝叶斯 P(d>0)=100%，HDI=[0.61, 1.77] 完全不含 0
- **结论**：信息整合效应被确证，这是最可靠的发现

### 2.3 先验稳健性验证

三种先验下 P(d>0)：

| 先验 | Full vs None | Shuffle vs None |
|------|-------------|-----------------|
| 怀疑 N(0,0.2) | 86.4% | 99.3% |

| 宽容 N(0,1.0) | 98.4% | 100.0% |

**Shuffle vs None** 在所有先验下都 >99%，结论先验稳健。
**Full vs None** 在最怀疑的先验下降至 86.5%，说明结论对先验有一定敏感性——这在小样本下是预期的。

## 3. 频率派 vs 贝叶斯：互补而非替代

| 维度 | 频率派 | 贝叶斯 |
|------|--------|--------|
| 多重比较校正后 | Full vs None 不显著 | P(d>0)=96.7% 仍确证 |
| 信息量 | p 值是二值判断 | 后验分布提供完整不确定性量化 |
| 小样本表现 | 依赖大样本渐近性 | 不假设渐近性，n=15 可靠 |
| 可解释性 | "p=0.037" 难直观 | "有效概率 96.7%" 直观 |

**互补关系**：频率派 p 值在多重比较校正后偏保守（Full vs None 不显著），贝叶斯方法通过完整后验分布提供了更细致的不确定性量化——两者一致指向"治理有效但仍有不确定性"。

## 4. 方法学局限

1. **似然函数为正态近似**：Cohen's d 的抽样分布用 N(d, σ_d) 近似，其中 σ_d = √(1/n₁ + 1/n₂)。n=15 下此近似合理但非精确。更严格的方法应使用非中心 t 分布作为似然。

2. **网格近似精度**：使用 10000 点网格，精度足够但不如 MCMC。对单参数推断，网格近似的结果与 MCMC 一致。

3. **先验选择的主观性**：Normal(0, 0.5) 是弱信息先验的常见选择（参考 Kruschke 2013, BEST 方法），但先验选择本质上带有主观性。通过三种先验的敏感性分析来缓解。

4. **HDI 下界略低于 0**：Full vs None 的 95% HDI = [-0.04, 1.13]，下界 -0.03 接近 0 但不为正。这说明"效应为正"的概率虽高（96.7%），但"效应为正且足够大"的确定性不足。

5. **多重比较的依赖性**：Full vs None (τ) 和 Full vs None (Q) 用同一批数据，高度相关。Bonferroni 校正假设独立，会过于保守。更精确的校正应考虑相关性（如 Holm-Bonferroni 或 FDR），但 Bonferroni 是最保守的报告方式。

---

## 5. 对竞赛和论文的建议

### 可用于报告的结论

> 治理效应的贝叶斯后验分析（Normal(0,0.5) 先验）显示：
> - 治理有效的后验概率 P(d>0) = 96.7%，超过 95% 确证阈值
> - 效应量后验均值 d = 0.54，95% HDI = [-0.04, 1.13]
> - 信息整合效应（shuffle vs none）在所有先验下 P(d>0) > 99%，确证成立
>
> **关于后验均值与样本均值差异**：样本 Cohen's d = 0.84，后验均值 d = 0.54。差异来自贝叶斯向先验均值 0 的收缩（shrinkage）——这是正态先验在小样本下的预期行为，并非计算错误。收缩程度取决于先验尺度（scale 越小收缩越大）。样本 d=0.84 是点估计，后验 d=0.54 是考虑了先验不确定性后的收缩估计，两者互补而非矛盾。
> **似然函数**：使用 Hedges & Olkin (1985) 的 d 抽样方差近似 σ²_d = (n1+n2)/(n1·n2) + d²/(2(n1+n2))，比仅用第一项更精确（大 d 时差异约 6-13%）。
>
> 频率派置换检验：Full vs None p=0.037（未校正显著），Bonferroni 校正后 p=0.110（不显著）；shuffle vs none p=0.0002（校正后仍显著）。
>
> 两种框架互补：贝叶斯在小样本下提供更细粒度的不确定性量化，频率派在多重比较校正下偏保守。

### 不可夸大的边界

- **不能说**"治理效应已被统计确认"——HDI 下界略低于 0
- **不能说**"贝叶斯证明治理有效"——应说"后验概率支持"
- **可以说的**是"贝叶斯后验 P(d>0)=96.7%，结合频率派 p=0.037，证据方向一致指向治理有效"

### 仍需扩样的理由

贝叶斯分析**不能替代扩样**。当前结论仍有不确定性（HDI 下界 -0.03），扩样到 n=30 可以：
- 收窄 HDI 区间
- 提高 Bonferroni 校正后的频率派显著性
- 确认效应量估计

**优先级调整**：扩样从 P0 降为 P1——贝叶斯分析已经提供了"96.7% 后验概率"可用于报告，扩样的紧迫性降低，但仍应进行以消除剩余不确定性。

## 5. 关键差距分析

### 5.1 ~~最大盲区：Inter-Agent Misalignment（FC2，32.3%）~~ 已部分填补（A3）

MAST 数据显示 FC2 是第二大失败类别。A3 实现前本项目**零覆盖**，现已通过 3 个新检测器填补 FM-2.4/2.5/2.6（合计 17.2%）。剩余 FC2 盲区：

- `referencedAgents[]` 字段 → ✅ `ignored_input` 检测器（FM-2.5）
- `itemBeliefs` 字段 → ✅ `reasoning_action_mismatch` 检测器（FM-2.6）
- `evidence[]` 字段 → ✅ `information_withholding` 检测器（FM-2.4）

**关键工程决策**：补 FC2 检测器不需要改 prompt，只需要在观测层加规则。A3 实现验证了这一判断——3 个检测器复用现有 MessageInfo 字段，零 prompt 改动，零新干预类型（均复用 force_reflection）。

### 5.3 本项目的独有优势

- **FM-1.5 Unaware of stopping**：MAST 论文将其列为 2.2%，但这是 MAS 工程痛点。SwarmAlpha 的热力学终止判定（R/T/H/F + TerminationDecider）是**MAST 论文未提供的解决方案**。
- **FM-3.1 Premature termination**：SwarmAlpha 的 `premature_consensus` 检测器直接对应此模式，且配合淬火态干预（注入多样性）形成闭环。

## 5. 安全信号接口建议（基于 ASI 对齐）

如未来实现安全钩子（用户原方案的 Task 3），SecuritySignal.type 应按以下映射：

| SwarmAlpha 检测场景 | OWASP ASI 类别 | SecuritySignal.type |
|---|---|---|
| 恶意 prompt 注入迹象 | ASI01 | `goal_hijack` |
| Agent 信念异常强化 | ASI10 | `rogue_agent` |
| 依赖链附带损害 | ASI08 | `cascading_failure` |
| Agent 隐藏独有信息 | ASI06（部分） | `context_poisoning` |
| Agent 间通信异常 | ASI07（部分） | `insecure_communication` |

**注意**：用户原方案的 `jailbreak_attempt`/`pii_leakage`/`budget_exceeded`/`tool_misuse` 等类型**不适用**于本项目——本项目 agent 无工具、无 PII、无预算系统。应改为上表的 5 类。

### 5.3 5 类信号的检测逻辑草案

| 信号类型 | 检测逻辑草案 | 数据来源 |
|---|---|---|
| `goal_hijack` | reasoning 与 taskDescription 相似度 < 阈值 | reasoning + task |
| `rogue_agent` | agent 信念连续 N 轮朝同一方向强化且超出群体均值 ±2σ | beliefHistory |
| `cascading_failure` | 依赖链下游 agent 信念变化与上游恶意 agent 相关性 > 0.7 | dependencyGraph + beliefHistory |
| `context_poisoning` | 复用 A3 `information_withholding` 检测器 | evidence[] |
| `insecure_communication` | 复用 A3 `ignored_input` 检测器 + 引用频率异常 | referencedAgents[] |

**关键复用**：A3 已实现的 FM-2.4/2.5 检测器可直接映射为 `context_poisoning`/`insecure_communication` 信号，无需重复开发。

## 6. 与 MAST 对齐的关系

| 维度 | MAST | OWASP ASI |
|---|---|---|
| 性质 | 失败模式分类法（描述性） | 安全风险分类法（规范性） |
| 关注点 | 系统为什么会 fail | 系统为什么会被攻击 |
| 与本项目关系 | 评估检测器覆盖度 | 评估实验场景覆盖度 |
| 覆盖率 | 39%（5.5/14，A3 后） | 40%（4/10 加权） |

**互补性**：MAST 评估"我们检测多少种失败"，OWASP ASI 评估"我们研究多少种攻击"。两者结合给出项目的**双重定位**：

> SwarmAlpha 是一个讨论型 LLM MAS 治理实验框架，覆盖 MAST 14 模式中的 39% 与 OWASP ASI 10 风险中的 40%，专注于 rogue agent（ASI10）与 cascading failures（ASI08）的量化分析。
