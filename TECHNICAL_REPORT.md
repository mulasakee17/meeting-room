## 1. Introduction

### 1.1 问题：认知治理层的缺失

LLM 多 Agent 系统正从研究原型转向生产部署。AutoGen、CrewAI、LangGraph 等框架协调 AI Agent 团队完成金融分析、临床决策、法律推理。然而这些系统继承了人类群体的集体决策缺陷：回声室、权威偏误、群体极化、过早共识。

2026 年涌现的"Agent 治理"工具——Microsoft Agent Governance Toolkit、Agent Control Standard (ACS)、NVIDIA OpenShell——全部聚焦**安全层**（未授权工具调用、预算超支、数据泄露）。**认知层**（检测 Agent 讨论何时偏向集体失败）缺乏生产级工具。

SwarmAlpha 填补这一空白：一个可嵌入的认知治理运行时，观察、检测、干预多 Agent 系统的集体决策失败。

### 1.2 研究问题

本报告围绕三个递进的研究问题：

| # | 研究问题 | 状态 |
|---|---------|------|
| RQ1 | 认知治理干预是否提升多 Agent 决策质量？ | ✅ Crisis 任务显著有效（d=0.92, p=0.005） |
| RQ2 | 社会热力学 F 分解能否为干预优先级提供理论框架？ | ⚠️ 诊断价值已验证，运行时排序价值未验证 |
| RQ3 | F 分解驱动的干预排序相比固定排序是否提升决策质量？ | ❌ A/B 对照未支持（d_z=−0.354, p=0.378） |

### 1.3 核心贡献

1. **治理有效性实证**：修复 4 个认知缺陷（D1-D4）后，Crisis 任务上治理显著提升 τ（d=0.92, p=0.005, power=88%）
2. **社会热力学诊断框架**：F 分解提供了分析框架，发现并修正了 force_reflection 映射的错误假设（H1 证伪，p=0.041）
3. **诚实的负面发现**：F 分解排序的 A/B 对照实验未显示显著改善，记录了边界条件
4. **范式批判**：识别多 Agent 讨论范式的 4 个认知缺陷（D1-D4），揭示治理环路断裂问题

## 2. 实验发现

### 2.1 实验设计

**2×2 因子设计**，两个任务 × 多个消融条件：

| 任务 | 难度 | 特征 | 样本量（闭环） |
|------|------|------|--------------|
| **Crisis Response** | 困难 | 5 个专业角色，隐藏信息需交叉验证 | 72 实验 |
| **Supplier Selection** | 简单 | 5 个采购角色，信息较透明 | 89 实验 |

**消融条件**：
- `none`：无治理基线
- `full`：完整治理（4 检测器 + F 分解排序 + 全部干预）
- `full_fixed`：完整治理但固定排序（A/B 对照 B 组）
- `shuffle`：打乱角色-信息绑定（结构干预，非认知干预）
- `full_diversity` / `full_weight` / `full_reflection` / `full_continue`：单干预消融

**评估指标**：Kendall's τ-b（agent 集体排序与 ground truth 排序的等级相关，tie 修正）

### 2.2 核心结果：治理有效性

| 任务 | 基线 τ (none) | 治理 τ (full) | Δτ | Cohen's d | p-value | Power |
|------|-------------|-------------|-----|-----------|---------|-------|
| **Crisis** | 0.41 | 0.62 | +0.21 | **0.92** | **0.005** | 88% |
| **Supplier** | 0.68 | 0.73 | +0.05 | 0.47 | 0.089 | 43% |

**Crisis 任务**：治理显著有效。τ 提升 51%，大效应量（d=0.92），统计功效充足（88%）。

**Supplier 任务**：方向一致但未达显著。天花板效应——基线 τ=0.68 已接近 full τ=0.767，治理空间有限。

### 2.3 Shuffle 边界条件

Shuffle 干预（确定性 +2 轮转 agent 已知信息，角色标签固定）揭示了一个关键边界条件：

| 任务 | 基线 τ | Shuffle τ | Δτ | Cohen's d | p-value |
|------|--------|----------|-----|-----------|---------|
| **Crisis**（困难） | 0.41 | 0.72 | +0.31 | **1.44** | <0.001 |
| **Supplier**（简单） | 0.68 | 0.69 | +0.01 | 0.09 | 0.78 |

**关键发现**：打破角色-信息绑定在困难任务上产生巨大改善（d=1.44），但在简单任务上无效。原因是天花板效应——Supplier 基线已接近最优，结构干预无额外空间。

**理论含义**：多 Agent 共识的相变边界不仅由交互参数（温度、从众压力）决定，还由角色-信息分配的拓扑结构根本塑造——这是当前统计物理模型缺失的维度。

### 2.4 共识与正确性的解耦

Kendall's τ（共识度）与决策质量的相关性：r ≈ −0.14（两个任务均如此）。

**含义**：共识水平不是正确性的可靠代理——高共识可能是"集体一致地错"（虚假共识）。这构成了 F 分解框架中 `introduce_diversity` 干预的理论基础，同时也暴露了 F 分解的根本盲区（见 §3.3）。

### 2.5 历史对照：165 环路断裂实验

在 D1-D4 修复前，收集了 165 个实验。这些实验中状态修改干预（reduce_weight、force_reflection）对 agent 不可见，治理环路断裂。这些数据仅作对照保留，不作为核心结论依据。

**161 闭环实验 vs 165 历史实验的关系**：
- 161 = Crisis 72 + Supplier 89，D1-D4 修复后收集，治理环路完整
- 165 = D1-D4 修复前收集，干预未实际到达 agent 感知，仅作"治理环路断裂"的证据

## 3. F 分解框架

### 3.1 理论基础：社会热力学

将多 Agent 系统类比为热力学系统，提供 3 个状态变量：

| 变量 | 物理含义 | 系统含义 | 计算来源 | 范围 |
|------|---------|---------|---------|------|
| **R**（序参量） | Kuramoto 同步度 | agent 信念方向同步程度 | `computeKuramotoOrder` | [0,1] |
| **T**（温度） | 热运动幅度 | 信念分散程度（标准差） | `normalizeTemperature(computeStd)` | [0,1] |
| **H**（熵） | 信息无序度 | 信念分布的不确定性 | `shannonEntropy`（5 bins, 归一化） | [0,1] |

**Helmholtz 自由能**：

$$F = U - TS = (1-R) + T \cdot H$$

- **(1−R)**：结构性无序——agent 信念方向未对齐
- **T·H**：热性无序——agent 信念既分散又分布广

这两个分量**正交**：可以同时高（既极化又噪声大）、同时低（真共识）、或一高一低（伪共识/伪极化）。

### 3.2 Kuramoto 序参量的关键修复（H4）

**原缺陷**：相位映射 `θ = π·b` 使 b=±0.99 都落在单位圆左侧附近，R≈1，将极化误判为共识。

**修复**：改为 `θ = (π/2)·b`：
- b=+1（强支持）→ θ=+π/2（单位圆顶部）
- b=−1（强反对）→ θ=−π/2（单位圆底部）
- 两者正对，R≈0（正确反映极化）

代码位置：[src/lib/governance/index.ts:532](./src/lib/governance/index.ts#L532)

### 3.3 干预-无序映射：四个可证伪假设

F 分解的核心设计：每种干预类型对应一种"无序病机"，F 分解告诉我们当前系统主要患哪种病。

| 干预类型 | 评分公式 | 设计假设（可证伪） | 回测状态 |
|---------|---------|-------------------|---------|
| `force_reflection` | `thermal·(1−structural)`（原 `structural`，H1 证伪后修正） | 原假设：反思对齐方向。**证伪**后修正为：反思是降噪干预 | ✅ 证伪→修正 |
| `reduce_weight` | `thermal`（即 T·H） | 假设权重压制抑制高噪 agent | ⚠️ 方向支持不显著 |
| `introduce_diversity` | `R·(1−H)` | 假设"高同步低熵"= 虚假共识 | ⏳ 未回测 |
| `continue_discussion` | `R·(1−H)·(1−F)` | 假设有序+低F+早期=过早收敛 | ❌ 证伪+禁用 |

**关键诚实声明**：这 4 个公式是基于干预机制的作用假设**设计**的，不是从理论**推导**的。每个公式背后是一个**可证伪的假设**，而非数学必然。

### 3.4 根本盲区：F 分解无法区分真/假共识

`R·(1−H)` 对"真共识"和"虚假共识"给出相同评分——两者都是高 R 低 H。区分需要 ground truth 或外部信号，F 分解本身做不到。这是三层架构中层 3（增量评估反馈）要解决的核心问题。

### 3.5 三层联合架构

| 层 | 功能 | 状态 |
|----|------|------|
| **层 1** | F 分解驱动干预优先级排序 | ✅ 已实现 |
| **层 2** | 任务难度感知门控（τ₁ + ΔF 轨迹调剂量） | 🔄 实验室 |
| **层 3** | 在线干预效果反馈（ΔT 自动降权有害干预） | 🔄 实验室 |

**层 2 原设计推翻**：原设计"F 低→收紧阈值"在天花板情形下会触发更多干预，加剧有害干预。推翻后改为任务难度感知门控：τ₁ 高 → dosage_scale 低 → 抑制干预。

## 4. 证伪与修正

### 4.1 回测方法论

**数据**：161 闭环实验中每次 force_reflection / reduce_weight 干预事件。

**方法**：按干预时系统的 (1−R) vs T·H 比值分桶，对比结构性主导组 vs 热性主导组的 Δτ（下一轮 tau − 当前轮 tau）。

**统计**：配对置换检验（5000 perms，(count+1)/(nPerms+1) 修正）+ Cohen's d。

**局限声明**：观察性研究，非因果确证（agent 在不同 F-state 非随机分配，存在混杂）。

### 4.2 假设 1（force_reflection ↔ structural）：证伪并修正

**原假设**：force_reflection 是对齐方向的干预，应在结构性无序主导时有效。

**回测结果**（97 事件）：

| 组 | n | 平均 Δτ |
|----|---|--------|
| 结构性主导 (1−R > T·H) | 59 | **−0.033**（有害） |
| 热性主导 (T·H ≥ 1−R) | 38 | **+0.115**（有益） |

- 置换检验 p = 0.041
- Cohen's d = −0.49

**结论**：**假设 1 证伪**。force_reflection 在结构性主导（极化）时有害，在热性主导时有益。force_reflection 不是对齐方向干预，而是降噪干预。

**修正**：评分公式从 `structural` 改为 `thermal·(1−structural)`——在热性主导且非极化时优先。

代码位置：[src/lib/governance/index.ts:567](./src/lib/governance/index.ts#L567)

### 4.3 假设 2（reduce_weight ↔ thermal）：方向支持但不显著

**原假设**：reduce_weight 是热性无序干预，应在热性主导时有效。

**回测结果**（85 事件）：

| 组 | n | 平均 Δτ |
|----|---|--------|
| 热性主导 | 67 | +0.182 |
| 结构性主导 | 18 | +0.067 |

- 置换检验 p = 0.100（未达 0.05）
- Cohen's d = +0.448

**结论**：**方向支持但不显著**。热性主导时 Δτ 更高，方向一致，但未达统计显著。可能是样本不均衡（67 vs 18）拉低检验力。

**诚实定性**：不能称为"证实"，仅"方向一致"。

### 4.4 假设 3（introduce_diversity ↔ 虚假共识）：未回测

**未回测原因**：echo chamber 检测器在 mock 测试中难以触发，需真实 LLM 消息数据。

**根本盲区**：即使回测，F 分解也无法区分"真共识"与"虚假共识"——两者都是高 R 低 H。这是层 3 要解决的问题。

### 4.5 假设 4（continue_discussion ↔ 过早收敛）：证伪并禁用

**实验结果**：Crisis 任务上 0% 有效率，平均 Δτ = −0.400（有害）。

**结论**：**假设 4 证伪**。continue_discussion 已默认禁用（`disabledInterventions`）。

### 4.6 证伪的价值

4 个假设中，1 个证伪并修正，1 个方向支持但不显著，1 个未回测，1 个证伪并禁用。这个证伪过程本身是 F 分解框架的核心价值——它提供了**结构化的、可证伪的分析框架**，使假设错误能被发现和修正。

## 5. A/B 对照实验

### 5.1 实验动机

§4 的回测是观察性研究，无法孤立 F 分解排序对决策质量的因果贡献。为回答"F 分解排序是否比固定排序更好"，设计了预注册的 A/B 配对对照实验。

### 5.2 实验设计

**预注册假设 H_F**：F 分解排序的 Δτ 显著高于固定排序（单尾）。

**配对设计**：
- A 组：现有 `crisis_full` 实验（F 分解排序，sortingMode='fdecomposition'）
- B 组：新增 `crisis_full_fixed` 实验（固定排序，sortingMode='fixed'）
- 配对：同 runIndex（相同 seed），A/B 仅在排序逻辑上不同

**固定排序定义**：保持检测器触发顺序（reduce_weight → introduce_diversity → force_reflection → continue_discussion），不按系统状态重排。代表"未引入 F 分解前"的现实基线。

**统计方法**：
- 主检验：配对置换检验（符号翻转，10000 perms，(count+1)/(nPerms+1) 修正）
- 效应量：Cohen's d_z = mean(Δτ_A − Δτ_B) / std(Δτ_A − Δτ_B)
- CI：t 分布配对均值差的 95% CI
- 双分析：ITT（所有配对）+ per-protocol（仅排序实际改变的配对）

**Pilot 策略**：先跑 n=8 配对，根据 d_z 决定是否扩样。

### 5.3 Pilot 结果（n=8，Crisis 任务）

| 指标 | ITT（全部 8 配对） | Per-protocol（8/8 排序改变） |
|------|-------------------|---------------------------|
| A 组 τ 均值（F 分解） | 0.6250 | 0.6250 |
| B 组 τ 均值（固定排序） | 0.6750 | 0.6750 |
| 配对差 Δτ_A − Δτ_B | **−0.0500** | **−0.0500** |
| Cohen's d_z | **−0.354** | **−0.354** |
| 配对置换检验 p-value | 0.3781 | 0.3765 |
| 95% CI | [−0.168, +0.068] | [−0.168, +0.068] |

**配对明细**：

| runIndex | τ_A (F 分解) | τ_B (固定) | diff | 排序不同 |
|----------|-------------|-----------|------|---------|
| 0 | 0.600 | 0.600 | 0.000 | 是 |
| 1 | 1.000 | 1.000 | 0.000 | 是 |
| 2 | 0.200 | 0.400 | −0.200 | 是 |
| 3 | 0.800 | 1.000 | −0.200 | 是 |
| 4 | 1.000 | 0.800 | +0.200 | 是 |
| 5 | 0.600 | 0.600 | 0.000 | 是 |
| 6 | 0.400 | 0.400 | 0.000 | 是 |
| 7 | 0.400 | 0.600 | −0.200 | 是 |

### 5.4 结论：H_F 不支持

**H_F 不支持**。8/8 配对中 F 分解确实改变了排序顺序（per-protocol 排除率 0%），但**改变排序没有改善 τ**，方向上反而略差（d_z=−0.354）。

- 4/8 配对 τ 相同（排序改变但结果不变）
- 3/8 固定排序更优（diff=−0.2）
- 1/8 F 分解更优（diff=+0.2）

**停止扩样决策**：基于预注册决策规则（d_z<0.2 或方向反转则停止），pilot 已显示方向反转，扩样无法翻转结论。

### 5.5 为什么未改善——三个可能解释

1. **3 轮太短**：同轮内多个干预都会执行（只是顺序不同），排序对最终 τ 影响有限——解释了 4/8 配对 τ 完全相同

2. **H1 修正后与固定排序重合**：force_reflection 降权后，F 分解和固定排序在 Crisis 上都把 reduce_weight 排前面，效果重合

3. **固定排序恰好接近最优**：Crisis 上 authority bias 和 polarization 最常见，固定排序 reduce_weight 优先恰好符合热性主导场景

### 5.6 F 分解价值的修正定性

| 维度 | F 分解贡献 | 证据强度 |
|------|-----------|---------|
| 诊断价值（发现 H1 错误） | ✅ 实质贡献 | 强（p=0.041） |
| 统一多检测器优先级（架构） | ✅ 架构合理 | 弱（无 Δτ 改善证据） |
| 提升决策质量（Δτ） | ❌ 未验证 | A/B 对照 d_z=−0.354，方向反转 |

**结论**：F 分解的主要价值是**诊断性**的——它提供了分析框架帮助发现 force_reflection 映射的错误假设。作为运行时干预排序机制，在 Crisis 任务 3 轮讨论中相比固定排序没有显著改善。更长讨论轮次或多任务场景下是否有效，留实验室验证。

## 6. 范式批判

### 6.1 多 Agent 讨论范式的四个认知缺陷（D1-D4）

在实验过程中，诊断出多 Agent 讨论范式存在 4 个根本认知缺陷，导致治理环路断裂：

| # | 认知缺陷 | 症状 | 影响 | 修复 |
|---|---------|------|------|------|
| **D1** | **状态感知缺失** | `buildPrompt` 未注入 agent 当前 belief/confidence，状态修改干预对 LLM 不可见 | reduce_weight、force_reflection 等干预无效 | buildPrompt 注入当前 belief/confidence |
| **D2** | **无对话历史** | 所有 agent 看同一全局摘要，不知道自己之前说过什么 | agent 无法维持立场一致性 | 个性化记忆：注入自己的历史发言和 @提及 |
| **D3** | **同步轮次（并行"念稿"）** | `Promise.all` 让所有 agent 并行发言 | agent 看不到同轮已发言的同伴 | 改为顺序 for 循环 |
| **D4** | **虚构影响力网络** | 同意/反对/说服边从信念数值差异**推断** | 产生虚假影响力幻影网络 | 仅从显式 referencedAgents 构建边 |

### 6.2 治理环路断裂的后果

D1-D4 导致 165 个历史实验中，状态修改干预**从未到达 agent 感知**：
- D1 使 reduce_weight/force_reflection 不可见（agent 感觉不到权重变化）
- D2 使 agent 无法跨轮维持立场（每轮"失忆"重新发言）
- D3 使同轮 agent 互不可见（并行念稿而非真正讨论）
- D4 使影响力图谱是虚构的（基于数值差而非真实引用）

**后果**：165 历史实验的"治理无效"结论是**环路断裂的假象**，不是治理本身无效。修复 D1-D4 后的 161 闭环实验显示 Crisis 任务治理显著有效（d=0.92）。

### 6.3 对范式的批判性思考

**当前多 Agent 讨论范式的根本问题**：多数框架假设"让 agent 交流就能涌现好的集体决策"，但忽略了：

1. **状态可见性**：agent 必须能感知自身状态变化，否则干预无效
2. **个体记忆**：agent 必须知道自己说过什么，否则无法维持立场
3. **顺序交互**：真正的讨论需要顺序发言，并行"念稿"不是讨论
4. **真实引用**：影响力网络必须基于显式引用，数值推断会产生幻影

**启示**：多 Agent 系统的"涌现"质量取决于底层交互机制的正确性。在断裂的范式上做实验，结论是不可信的——这正是 165 历史实验的教训。

### 6.4 统计方法的诚信修正

在实验过程中发现并修正了多个统计方法问题：

| 问题 | 原做法 | 修正 |
|------|--------|------|
| p=0.000 假阳性 | 置换检验无修正 | (count+1)/(nPerms+1) 修正 |
| τ-a 忽略 ties | 使用 τ-a | 改用 τ-b（tie 修正） |
| t 分布表不完整 | 线性插值 | 补全 1-30 整数值 |
| BH FDR 非标准 | 简单阈值 | 标准步降程序 |
| Welch p 值小样本偏差 | 无修正 | t_critical/z_critical 修正因子 |

## 7. 下一步方向

### 7.1 实验室优先级

| 优先级 | 任务 | 预期价值 |
|--------|------|---------|
| **P0** | 扩样 Crisis full_fixed 到 n=24，确认 A/B 对照结论 | 验证负面发现是否稳定 |
| **P1** | 层 2 任务难度感知门控实现与验证 | 解决 Supplier 天花板效应 |
| **P1** | 层 3 在线干预效果反馈实现 | 解决真/假共识盲区 |
| **P2** | H3 回测（introduce_diversity ↔ 虚假共识） | 需 echo chamber 真实触发数据 |
| **P2** | 更长轮次（5/7 轮）A/B 对照 | 验证 F 分解在长讨论中是否有效 |
| **P3** | InterventionType 闭集类型扩展 | 当前 4 种干预类型限制扩展性 |

### 7.2 理论方向

1. **F 分解的诊断价值深化**：F 分解作为分析框架已证明有效（发现 H1 错误），但作为运行时排序机制未验证。需要理论分析为何在 3 轮讨论中排序影响有限——可能是"同轮干预集合相同时，顺序对最终收敛轨迹的二阶效应"。

2. **真/假共识区分**：F 分解无法区分真共识与虚假共识。层 3 的在线 Δτ 反馈是一种经验性学习方案，但理论上的 ground truth-free 区分方法仍需探索。

3. **角色-信息拓扑**：shuffle 实验（d=1.44）显示角色-信息绑定是相变边界的结构前兆。这指向一个新理论方向：多 Agent 共识的质量不仅由交互动力学决定，还由初始信息分配的拓扑结构根本塑造。

### 7.3 方法学方向

1. **因果推断强化**：当前回测是观察性研究。理想方案是随机化干预排序（A/B 对照已部分实现），但完整因果推断需要更精细的实验设计。

2. **跨任务泛化验证**：当前结论基于 Crisis + Supplier 两任务。需要更多任务类型（如弱依赖任务、强依赖任务）验证治理有效性的边界条件。

3. **成本-效益分析**：CustomAgent 已追踪 token 用量、API 调用数、延迟。需要系统分析治理的边际成本与边际收益。
